# 情绪向量化最佳策略分析

## 📊 当前系统分析

### 现有架构
- **向量数据库**：ChromaDB（使用 text2vec-chinese，768维）
- **存储方式**：
  - `documents`：文本内容（会被向量化）
  - `metadatas`：结构化数据（用于过滤，不会被向量化）
- **检索方式**：
  - 语义相似度检索（基于documents的向量）
  - Metadata过滤（where条件）

### 情绪数据现状
- ✅ **PostgreSQL中存储**：`CharacterState`表包含所有情绪状态值
- ✅ **AI生成时使用**：情绪值用于生成对话的prompt
- ❌ **向量数据库中缺失**：未存储到metadata或documents中
- ❌ **无法向量检索**：无法基于情绪状态检索相似历史事件

---

## 🎯 情绪向量化的目标

1. **语义检索**：能够基于情绪状态检索相似的历史事件
2. **精确过滤**：能够按情绪范围过滤数据
3. **上下文连贯**：新事件生成时能参考相似情绪状态的历史
4. **性能优化**：不影响现有检索性能

---

## 💡 策略方案对比

### 方案1：纯Metadata存储 ⚠️ 不推荐

**实现方式**：
```python
metadata = {
    'emotion': states.emotion,
    'favorability': states.favorability,
    # ... 其他状态值
}
```

**优点**：
- ✅ 实现简单
- ✅ 可以精确过滤（where条件）
- ✅ 不影响documents文本质量

**缺点**：
- ❌ **metadata不会被向量化**（ChromaDB特性）
- ❌ 无法用于语义相似度检索
- ❌ 无法找到"情绪相似"的历史事件

**结论**：❌ **不满足向量化需求**

---

### 方案2：纯Documents文本 ⚠️ 部分推荐

**实现方式**：
```python
emotion_text = f"""
情绪状态：情绪值{emotion}，好感度{favorability}，信任度{trust}，敌意{hostility}
"""
content = f"{story_background}\n\n{emotion_text}\n\n{dialogue_text}"
```

**优点**：
- ✅ 会被向量化
- ✅ 可以语义检索
- ✅ 实现简单

**缺点**：
- ❌ 数字值在向量空间中意义不明确（"情绪值50"和"情绪值51"语义相似度低）
- ❌ 可能影响文本的语义检索质量
- ❌ 无法精确过滤

**结论**：⚠️ **可用但不理想**

---

### 方案3：情绪标签化 + Documents ⭐⭐⭐ 推荐

**实现方式**：
```python
def get_emotion_tags(states):
    """将情绪状态转换为语义标签"""
    tags = []
    
    # 情绪标签
    if states.emotion > 70:
        tags.append("情绪高涨 心情愉悦")
    elif states.emotion > 40:
        tags.append("情绪稳定 心情平静")
    elif states.emotion > 20:
        tags.append("情绪低落 心情不佳")
    else:
        tags.append("情绪极差 心情沮丧")
    
    # 好感度标签
    if states.favorability > 70:
        tags.append("对玩家非常有好感 关系亲密")
    elif states.favorability > 40:
        tags.append("对玩家有好感 关系友好")
    elif states.favorability < -30:
        tags.append("对玩家有敌意 关系紧张")
    
    # 信任度标签
    if states.trust > 70:
        tags.append("高度信任 愿意分享")
    elif states.trust < 30:
        tags.append("缺乏信任 保持距离")
    
    return " ".join(tags)

# 添加到documents
emotion_tags = get_emotion_tags(states)
content = f"{story_background}\n\n[情绪状态：{emotion_tags}]\n\n{dialogue_text}"
```

**优点**：
- ✅ **语义化**：标签是自然语言，向量化效果好
- ✅ **可检索**：能找到"情绪高涨"的相似历史
- ✅ **不影响主文本**：标签独立，不影响对话语义
- ✅ **可扩展**：可以添加更多语义标签

**缺点**：
- ⚠️ 需要设计标签系统
- ⚠️ 标签粒度需要平衡（太细或太粗都不好）

**结论**：⭐⭐⭐ **最佳方案**

### 方案4：混合方案（Metadata + 标签Documents）⭐⭐⭐⭐ 最推荐

**实现方式**：
```python
# 1. Metadata存储精确值（用于过滤）
metadata = {
    'emotion': states.emotion,
    'favorability': states.favorability,
    'emotion_level': 'high' if states.emotion > 70 else 'medium' if states.emotion > 40 else 'low',
    # ... 其他状态值
}

# 2. Documents包含语义标签（用于向量检索）
emotion_tags = get_emotion_tags(states)
content = f"{story_background}\n\n[情绪状态：{emotion_tags}]\n\n{dialogue_text}"
```

**优点**：
- ✅ **双重保障**：既能精确过滤又能语义检索
- ✅ **灵活性高**：可以根据需求选择检索方式
- ✅ **性能优化**：可以先过滤再检索
- ✅ **向后兼容**：不影响现有功能

**缺点**：
- ⚠️ 需要维护两处数据
- ⚠️ 实现稍复杂

**结论**：⭐⭐⭐⭐ **最推荐方案**

```python
# backend/database/vector_db.py

def search_similar_events_by_emotion(self, character_id: int, emotion_level: str = None, 
                                     min_emotion: int = None, max_emotion: int = None,
                                     query: str = None, n_results: int = 3):
    """基于情绪状态检索相似事件"""
    where_clause = {"character_id": str(character_id)}
    
    # 情绪级别过滤
    if emotion_level:
        where_clause['emotion_level'] = emotion_level
    
    # 情绪值范围过滤
    if min_emotion is not None:
        where_clause['emotion'] = {'$gte': str(min_emotion)}
    if max_emotion is not None:
        where_clause['emotion'] = {'$lte': str(max_emotion)}
    
    # 查询文本（如果提供，会进行语义检索）
    query_text = query or "情绪状态 心情 感受"
    
    results = self.collection.query(
        query_texts=[query_text],
        n_results=n_results,
        where=where_clause
    )
    
    return results
```

## 📈 方案优势总结

### 混合方案的优势

1. **语义检索**：情绪标签会被向量化，可以找到"情绪高涨"的相似历史
2. **精确过滤**：metadata中的数值可以精确过滤
3. **灵活查询**：
   - 语义查询：`search_similar_events(character_id, "情绪高涨 心情愉悦")`
   - 精确过滤：`where={"emotion_level": "high"}`
   - 组合查询：先过滤再检索
4. **向后兼容**：不影响现有检索逻辑
5. **性能优化**：可以先过滤再检索，减少计算量

## 🎨 情绪标签设计建议

### 标签设计原则

1. **语义化**：使用自然语言，不要用数字
2. **层次化**：设计多个层次（高/中/低）
3. **组合化**：可以组合多个情绪维度
4. **一致性**：标签命名要一致

### 推荐标签体系

```
情绪维度：
- 情绪值：情绪高涨/情绪稳定/情绪低落/情绪极差
- 好感度：非常有好感/有好感/态度中性/有些反感/有敌意
- 信任度：高度信任/信任/缺乏信任
- 情感状态：非常快乐/非常悲伤/压力很大/焦虑不安

组合示例：
"情绪高涨 心情愉悦 积极乐观 对玩家非常有好感 关系亲密 高度信任玩家"
```
