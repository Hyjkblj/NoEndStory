# 游戏结局判断条件

## 概述

游戏进入结局需要满足两个条件：
1. **事件数量条件**：完成足够数量的事件
2. **状态值条件**：根据角色的状态值判定结局类型

---

## 一、游戏结束判断

### 判断方法
`StoryEngine.is_game_finished()` 

### 判断条件
```python
current_event_count > max_events
```

### 配置参数
- **`max_events`**: 默认值为 **15**（在 `config.py` 中配置）
- **总事件数**: 17（开头事件 1 + 中间事件 15 + 结尾事件 1）

### 说明
- 当完成的事件数量超过 `max_events` 时，游戏判定为结束
- 事件计数包括：
  - 开头事件（1个）
  - 中间事件（最多15个）
  - 结尾事件（1个，不计入计数）

---

## 二、结局类型判断

### 判断方法
`StoryEngine.get_ending_event()` 和 `GameService.check_ending()`

### 判断依据
根据角色的三个核心状态值：
- **好感度** (favorability): 0-100
- **信任度** (trust): 0-100
- **敌意值** (hostility): 0-100

### 结局类型及条件

#### 1. 美好结局 (Happy Ending / Good Ending)

**条件（`get_ending_event` 方法）**：
```
好感度 > 70  AND  信任度 > 60  AND  敌意值 < 20
```

**条件（`check_ending` 方法）**：
```
好感度 > 60  AND  信任度 > 50
```

**描述**：
> 经过一系列事件，你们的关系变得更加亲密，这是一个美好的结局。

---

#### 2. 坏结局 (Bad Ending)

**条件**：
```
好感度 < 30  OR  敌意值 > 50
```

**描述**：
> 关系走向了不好的方向，你们之间的距离越来越远。

---

#### 3. 中性结局 (Neutral Ending)

**条件**：
```
信任度 > 50  AND  好感度 > 40  AND  不满足美好结局条件
```

**描述**：
> 你们的关系保持在一个稳定的状态，未来还有无限可能。

---

#### 4. 开放结局 (Open Ending)

**条件**：
```
不满足以上任何条件
```

**描述**：
> 故事还在继续，结局尚未确定...

---

## 三、状态值说明

### 核心状态值

| 状态值 | 范围 | 说明 |
|--------|------|------|
| 好感度 (favorability) | 0-100 | 角色对玩家的好感程度 |
| 信任度 (trust) | 0-100 | 角色对玩家的信任程度 |
| 敌意值 (hostility) | 0-100 | 角色对玩家的敌意程度 |

### 其他状态值（不影响结局判断，但影响对话生成）

- 依赖度 (dependence)
- 情绪值 (emotion)
- 压力值 (stress)
- 焦虑值 (anxiety)
- 快乐值 (happiness)
- 悲伤值 (sadness)
- 自信值 (confidence)
- 主动性 (initiative)
- 谨慎度 (caution)

---

## 四、状态值变化机制

### 玩家选项影响

玩家在对话中选择不同的选项会影响状态值：

1. **积极选项**（选项1）：
   - 提升好感度和信任度
   - 降低敌意值

2. **中性选项**（选项2）：
   - 保持现状，不改变状态值

3. **消极选项**（选项3）：
   - 降低好感度和信任度
   - 可能提升敌意值

### 状态变化示例

```python
# 积极选项的状态变化
{
    'favorability': +5,
    'trust': +3,
    'hostility': -2
}

# 消极选项的状态变化
{
    'favorability': -5,
    'trust': -3,
    'hostility': +2
}
```

## 五、代码位置

### 关键文件

1. **`backend/game/story_engine.py`**
   - `is_game_finished()`: 判断游戏是否结束
   - `get_ending_event()`: 获取结局事件（根据状态值判定结局类型）

2. **`backend/api/services/game_service.py`**
   - `check_ending()`: 检查是否满足结局条件
   - `trigger_ending()`: 触发结局

3. **`backend/config.py`**
   - `GAME_CONFIG['max_events']`: 最大事件数配置

## 六、注意事项

### 1. 判断条件不一致

`get_ending_event()` 和 `check_ending()` 中美好结局的判断条件略有不同：

- **`get_ending_event()`**: 好感度 > 70 且 信任度 > 60 且 敌意值 < 20
- **`check_ending()`**: 好感度 > 60 且 信任度 > 50

**建议**：统一这两个方法的判断条件，确保一致性。

### 2. 事件计数

- 开头事件不计入 `current_event_count`
- 中间事件计入 `current_event_count`
- 结尾事件不计入 `current_event_count`
- 当 `current_event_count > max_events` 时触发结局

### 3. 状态值初始化

角色创建时，所有状态值初始化为默认值（通常为 50 或 0，取决于具体实现）。

---

## 七、修改建议

### 1. 统一判断条件

建议在 `config.py` 中定义结局判断的阈值：

```python
ENDING_CONDITIONS = {
    'happy_ending': {
        'favorability': 70,
        'trust': 60,
        'hostility': 20
    },
    'bad_ending': {
        'favorability': 30,
        'hostility': 50
    },
    'neutral_ending': {
        'favorability': 40,
        'trust': 50
    }
}
```

### 2. 可配置化

将 `max_events` 和结局判断条件都设为可配置项，方便调整游戏难度和长度。

## 八、测试建议

### 测试场景

1. **测试美好结局**：
   - 持续选择积极选项
   - 确保好感度 > 70，信任度 > 60，敌意值 < 20

2. **测试坏结局**：
   - 持续选择消极选项
   - 确保好感度 < 30 或 敌意值 > 50

3. **测试中性结局**：
   - 混合选择选项
   - 确保信任度 > 50，好感度 > 40，但不满足美好结局条件

4. **测试开放结局**：
   - 保持状态值在中间范围
   - 不满足任何明确结局条件

---

## 总结

游戏进入结局需要：
1. ✅ 完成至少 15 个中间事件（`current_event_count > 15`）
2. ✅ 根据最终状态值判定结局类型：
   - 美好结局：高好感度 + 高信任度 + 低敌意值
   - 坏结局：低好感度 或 高敌意值
   - 中性结局：中等好感度 + 高信任度
   - 开放结局：其他情况
