# 初遇事件生成逻辑详细说明

## 一、整体流程

```
用户选择初遇场景（如：学校）
    ↓
调用 initialize_story API
    ↓
GameService.initialize_story(thread_id, character_id, scene_id)
    ↓
StoryEngine.get_opening_event(character_id, scene_id)
    ↓
随机选择 opening_events 中的一个事件
    ↓
智能场景映射（事件ID → 场景ID）
    ↓
EventGenerator.generate_event() 生成事件（包含故事背景）
    ↓
生成第一轮对话
    ↓
生成场景图片并合成（场景图 + 人物图）
    ↓
返回给前端
```

## 二、详细步骤

### 步骤1: 获取初遇事件 (`get_opening_event`)

**位置**: `backend/game/story_engine.py`

```python
def get_opening_event(character_id, scene_id='school'):
    # 1. 从 SCENES 获取场景配置
    scene = SCENES.get(scene_id)
    
    # 2. 获取该场景的 opening_events 列表
    #    如果场景没有定义 opening_events，使用默认事件
    opening_events = scene.get('opening_events', [默认事件])
    
    # 3. 随机选择一个初遇事件
    selected_event = random.choice(opening_events)
    
    # 4. 智能场景映射（重要！）
    #    如果事件ID对应一个场景（如 cafeteria 事件ID对应 cafeteria 场景），
    #    则使用事件ID作为场景ID（用于场景图片显示）
    if event_id in SCENES and event_id != 'school':
        event_scene_id = event_id  # 使用事件ID作为场景ID
    
    # 5. 调用 EventGenerator 生成事件（包含故事背景）
    event = event_generator.generate_event(
        character_id=character_id,
        event_id=selected_event['id'],
        event_context=selected_event['description'],
        previous_contexts=[]
    )
    
    # 6. 设置事件属性
    event['title'] = selected_event['title']
    event['scene'] = event_scene_id  # 使用映射后的场景ID
    event['original_scene'] = scene_id  # 保存原始选择的场景
    
    # 7. 更新当前场景
    self.current_scene = event_scene_id
    
    return event
```

### 步骤3: 故事背景生成

**位置**: `backend/game/event_generator.py`

```python
def generate_story_background(character_id, event_context, previous_contexts):
    # 1. 从向量数据库检索相似历史事件（避免重复）
    results = vector_db.search_similar_events(
        character_id=character_id,
        query=event_context,
        n_results=5
    )
    
    # 2. 提取历史事件文本
    previous_events = results['documents']
    
    # 3. 调用AI生成故事背景（使用通义千问）
    story_background = ai_generator.generate_story_background(
        character_id=character_id,
        previous_events=previous_events,
        current_context=event_context
    )
    
    return story_background
```

### 步骤4: 生成场景图片

**位置**: `backend/api/services/game_service.py` (initialize_story方法)

```python
# 使用事件对应的场景ID生成场景图片
event_scene = event.get('scene', scene_id)

    'scene_name': scene_info.get('name', event_scene),
    'scene_description': scene_info.get('description', '')

scene_image_url = image_service.generate_scene_image(scene_data, event_scene)
```

**场景图片生成逻辑**:
1. 生成场景图片 prompt: `"生成一个XXX场景图片，二次元写实画风，图中无人物"`
2. 使用火山引擎或通义万相生成图片
3. 图片尺寸: **2560x1440** (16:9比例，满足最小像素要求 3,686,400)
4. 保存到本地: `backend/images/scenes/{scene_id}_{scene_name}.{ext}`

## 四、场景映射规则

### 规则1: 事件ID = 场景ID
如果 `opening_event['id']` 在 `SCENES` 中存在，且不是 'school'，则使用事件ID作为场景ID。

**示例**:
- `cafeteria` 事件 → 使用 `cafeteria` 场景
- `classroom` 事件 → 使用 `classroom` 场景
- `cafe_nearby` 事件 → 使用 `cafe_nearby` 场景

### 规则2: 事件ID ≠ 场景ID
如果 `opening_event['id']` 不在 `SCENES` 中，则使用原始选择的场景ID。

**示例**:
- `library_meet` 事件 → 使用 `school` 场景（原始选择）
- `deskmate` 事件 → 使用 `school` 场景（原始选择）
- `badminton_court` 事件 → 使用 `school` 场景（原始选择）

## 五、场景图片生成问题修复

### 问题
火山引擎要求图片至少 **3,686,400 像素**，但当前使用 `1920x1080` 只有 **2,073,600 像素**。

### 解决方案
将场景图片尺寸改为 **2560x1440**：
- 像素数: 2560 × 1440 = **3,686,400** ✓（正好满足要求）
- 比例: 16:9 ✓（保持原有比例）
- 分辨率: 2K ✓（高质量）

### 修改位置
`backend/api/services/image_service.py`:
- `_generate_scene_with_volcengine`: 将 `scene_image_size` 从 `"1920x1080"` 改为 `"2560x1440"`

## 六、完整数据流

```
前端: POST /api/v1/characters/initialize-story
    {
        "thread_id": "...",
        "character_id": "123",
        "scene_id": "school"
    }
    ↓
后端: GameService.initialize_story()
    ↓
1. StoryEngine.get_opening_event(character_id, "school")
   - 从 SCENES['school']['opening_events'] 随机选择
   - 假设选择: {'id': 'cafeteria', 'title': '食堂偶遇', ...}
   - 场景映射: event_scene_id = 'cafeteria'
   - 生成事件: event = {..., 'scene': 'cafeteria'}
    ↓
2. EventGenerator.generate_event()
   - 生成故事背景（使用AI）
   - event['story_background'] = "..."
    ↓
3. StoryEngine.get_next_dialogue_round()
   - 生成第一轮对话
   - dialogue_data = {'character_dialogue': "...", 'player_options': [...]}
    ↓

   - 合成场景图和人物图

    ↓
6. 返回给前端
    {

        "story_background": "...",

        "character_dialogue": "...",

    }
```

## 九、注意事项

1. **场景映射**: 事件ID如果对应场景ID，会自动切换场景，确保使用正确的场景图片
2. **图片尺寸**: 场景图片使用 2560x1440（满足火山引擎最小像素要求）
3. **图片格式**: 场景图片可以是 JPG/PNG，人物图片固定为 PNG（透明背景）
4. **合成逻辑**: 场景图作为背景，人物图叠加在上方（居中偏下）
