# 火山引擎（豆包）配置指南

## 概述

本项目已支持火山引擎豆包大模型，**仅用于图片生成**：
- **图片生成**：角色立绘生成

> **注意**：文本生成（故事背景、角色对话、玩家选项等）使用通义千问，不使用火山引擎。

---

## 一、准备工作

### 1. 注册火山引擎账号

1. 访问 [火山引擎官网](https://www.volcengine.com/)
2. 注册/登录账号
3. 完成实名认证

### 2. 创建API密钥（ARK API Key）

1. 登录火山引擎控制台
2. 进入 **豆包开放平台** 或 **API密钥管理**
3. 创建新的 **ARK API Key**（Bearer Token）
4. 保存 API Key（注意保密，这是Bearer Token，不是AK/SK）

### 3. 开通图片生成服务

1. 进入 **ImageX** → **豆包 AIGC**
2. 开通服务
3. 确认可用模型（如：`doubao-seedream-4-0-250828`）

## 三、配置环境变量

在 `backend\.env` 文件中添加以下配置：

```env
# 火山引擎配置（仅用于图片生成 - Seedream 4.0-4.5 API）
VOLCENGINE_ARK_API_KEY=你的ARK_API_KEY  # Bearer Token
VOLCENGINE_REGION=cn-beijing  # 区域：cn-beijing（推荐）
VOLCENGINE_IMAGE_MODEL=doubao-seedream-4-5-251128  # 图片生成模型
VOLCENGINE_IMAGE_SIZE=2K  # 图片尺寸：2K, 4K, 1024x1024等
```

### 配置说明

| 变量 | 说明 | 示例 |
|------|------|------|
| `VOLCENGINE_ARK_API_KEY` | ARK API Key (Bearer Token) | `xxxxxxxxxxxxxxxxxxxxx` |
| `VOLCENGINE_REGION` | 区域 | `cn-beijing`（推荐） |
| `VOLCENGINE_IMAGE_MODEL` | 图片生成模型 | `doubao-seedream-4-5-251128` 或 `doubao-seedream-4-0-250828` |
| `VOLCENGINE_IMAGE_SIZE` | 图片尺寸 | `2K`（推荐）、`4K`、`1024x1024` 等 |

> **注意**：
> - 使用 **ARK API Key**（Bearer Token），不是 AK/SK
> - 推荐使用 `doubao-seedream-4-5-251128`（最新版本）或 `doubao-seedream-4-0-250828`
> - 推荐尺寸：`2K`（2048x2048）或 `4K`（4096x4096）

## 四、服务优先级

### 文本生成服务

- **通义千问**（如果配置了 `DASHSCOPE_API_KEY`）
- **规则生成**（如果未配置通义千问）

> **注意**：文本生成不使用火山引擎，只使用通义千问。

### 图片生成服务

1. **火山引擎**（如果配置了 `VOLCENGINE_AK` 和 `VOLCENGINE_SK`）
2. **通义万相**（如果配置了 `DASHSCOPE_API_KEY`）
3. **不可用**（如果都未配置）

## 七、常见问题

### Q1: 提示"requests未安装"

**解决方案**：
```powershell
pip install requests
```

### Q2: 提示"火山引擎初始化失败"或"API请求失败"

**可能原因**：
1. ARK API Key 配置错误或无效
2. Region 配置错误
3. 图片生成服务未开通
4. API Key 权限不足

**解决方案**：
1. 检查 `.env` 文件中的 `VOLCENGINE_ARK_API_KEY` 配置
2. 确认 ARK API Key 正确（在控制台重新生成）
3. 确认图片生成服务已开通
4. 确认 API Key 有图片生成权限
5. 查看控制台详细错误信息

### Q3: 文本生成使用的是哪个服务？

**答案**：
文本生成始终使用通义千问（如果配置了 `DASHSCOPE_API_KEY`），不使用火山引擎。火山引擎仅用于图片生成。

### Q4: 图片生成失败

**可能原因**：
1. 图片生成服务未开通
2. 模型ID错误
3. Prompt 不符合要求

**解决方案**：
1. 检查火山引擎控制台是否开通了 ImageX-豆包 AIGC 服务
2. 确认 `VOLCENGINE_IMAGE_MODEL` 配置正确
3. 查看控制台错误信息

## 十、总结

配置火山引擎后：

1. ✅ **图片生成**：自动使用火山引擎图片生成服务
2. ✅ **文本生成**：继续使用通义千问（不受火山引擎配置影响）
3. ✅ **回退机制**：如果火山引擎图片生成不可用，自动回退到通义万相

配置完成后，重启后端服务即可生效。
