# 《烽火笔锋》抗战战地记者决策与沟通能力训练系统方案

## 1. 项目基本信息

- 项目名称：烽火笔锋
- 项目全称：1937年7月7日至1945年9月2日抗战战地记者决策与沟通能力训练系统
- 项目类型：人工智能实践赛（教育与文化方向）
- 项目定位：历史情境驱动的高风险沟通能力训练平台
- 版本：v1.0（方案整合版）

## 2. 项目摘要

本项目将现有 AI 剧情交互系统重构为“历史情境下的记者决策训练系统”。用户扮演战地记者，在抗战关键节点进行信息采集、事实核验、报道发布与风险沟通决策。系统通过知识追踪（KT）持续建模用户能力状态，动态推送下一轮训练场景，并给出可解释复盘报告，实现“可训练、可评估、可成长”的闭环。

本项目强调三点创新：

1. 固定史实时间轴，不娱乐化改史
2. 以能力成长为目标，不以剧情通关为目标
3. 结合 LLM 语义评估与 KT 量化追踪，形成可解释训练体系

## 3. 场景与痛点

### 3.1 应用场景

面向高校新闻传播、历史教育、思政课程和应急传播训练课程，提供“高压信息环境下的报道决策演练”。

### 3.2 核心痛点

1. 传统历史教学偏重记忆史实，缺少决策训练
2. 传播伦理教学偏理论，缺少高压对抗环境实操
3. 学员能力评估依赖主观印象，难量化与追踪
4. 训练内容缺少个性化路径，重复训练效率低

### 3.3 本项目解决的问题

1. 在信息不完整与时间压力下，如何做真实、及时、低伤害的报道决策
2. 如何量化学员在高风险沟通中的具体能力短板
3. 如何根据短板自动推荐下一轮最有效训练场景

## 4. 项目目标与边界

### 4.1 项目目标

1. 建立抗战历史节点的结构化训练场景库
2. 建立记者高风险沟通能力图谱与评分标准
3. 建立 KT 驱动的个性化训练与复盘体系
4. 输出可视化成长报告（雷达图、曲线、证据链）

### 4.2 非目标（边界）

1. 不篡改历史结论，不否认已确立史实
2. 不作为真实新闻发布工具
3. 不作为医疗或心理诊断系统

## 5. 时间轴与历史约束

系统固定时间范围为：

- 起点：1937年7月7日（卢沟桥事变）
- 终点：1945年9月2日（日本签署投降书）

约束原则：

1. 场景时间不得超出上述区间
2. 事件节点必须锚定历史事实
3. 分支仅用于“反事实训练”，不得改写历史主结论

## 6. 能力图谱（KT 跟踪对象）

定义 8 个核心能力维度（K1-K8）：

| 维度 | 能力名称 | 能力定义 | 典型失误 |
|---|---|---|---|
| K1 | 史实核验 | 时间、地点、事件事实核查能力 | 未核实即发布 |
| K2 | 来源可信评估 | 对消息源可靠性的分级判断 | 单一来源直接定论 |
| K3 | 时效-准确平衡 | 抢时效与保准确的平衡能力 | 一味追快或一味延迟 |
| K4 | 风险沟通表达 | 在不失真的前提下降低恐慌 | 情绪化标题扩散恐慌 |
| K5 | 伦理与安全边界 | 报道中对人员和线索保护能力 | 暴露线人、二次伤害 |
| K6 | 反谣与纠偏 | 对冲突信息的澄清与更正能力 | 错后不更正 |
| K7 | 公共行动指引 | 给出明确、可执行的建议能力 | 结论模糊、不可执行 |
| K8 | 沟通闭环管理 | 发布-反馈-更新的闭环能力 | 无后续更新机制 |

## 7. 技术路线（BKT + LLM-KT + DKT）

### 7.1 总体结构

1. 历史场景引擎：按时间轴与节点触发训练任务
2. 交互引擎：文本/语音多轮对话
3. LLM 评估器：抽取证据并做多维评分
4. KT 引擎：更新能力掌握度
5. 自适应策略器：选择下一轮场景
6. 可视化报告器：输出成长结果

### 7.2 阶段一：BKT（冷启动主力）

每个能力维度一个掌握度 \( P(L_k) \)，每轮根据观测更新：

- 设 slip 为已掌握但答错概率
- 设 guess 为未掌握但答对概率
- 设 T 为学习转移概率

更新逻辑：

1. 根据本轮观测（正确/错误）更新后验掌握度
2. 根据学习转移概率计算下一时刻掌握度

优点：小数据稳定、解释性强，适合项目早期落地和答辩展示。

### 7.3 阶段二：LLM-KT（语义评估增强）

LLM 不直接输出“总分”，而是输出结构化证据：

```json
{
  "skill_scores": {
    "K1": {"score": 0.72, "confidence": 0.81, "evidence": ["..."]},
    "K5": {"score": 0.34, "confidence": 0.76, "evidence": ["..."]}
  }
}
```

将 `score/confidence` 转换为 KT 观测值，提升自由文本场景下的评估质量。

### 7.4 阶段三：DKT（规模化提升）

在训练样本积累后，引入 DKT 建模能力间迁移关系，用于：

1. 校正 BKT 参数
2. 预测“下一题成功概率”
3. 优化场景推荐策略

## 8. 自适应训练策略

每轮场景选择目标函数：

`utility = α * 信息增益 + β * 风险优先级 + γ * 短板覆盖 - δ * 认知负荷`

其中：

1. 信息增益：当前场景对能力不确定性的降低程度
2. 风险优先级：高风险能力（如 K1/K5）权重更高
3. 短板覆盖：优先训练低掌握度维度
4. 认知负荷：避免连续高压导致训练失效

## 9. 六大核心训练场景

| 场景编号 | 时间锚点 | 训练任务 | 关键决策 |
|---|---|---|---|
| S1 | 1937-07-07 卢沟桥事变 | 首条快讯发布 | 抢发/核验后发/双轨发布 |
| S2 | 1937-08-13 至 1937-11-26 淞沪会战 | 连续战况报道 | 标题口径与伤亡表述 |
| S3 | 1937-12-13 前后南京失守 | 高冲击信息处理 | 是否公开线索细节 |
| S4 | 1938-06 至 1938-10 武汉会战 | 后方动员沟通 | 稳定情绪与事实平衡 |
| S5 | 1940-08 至 1940-12 百团大战 | 多源冲突信息核验 | 单源定论或多源校验 |
| S6 | 1945-08-15 与 1945-09-02 终战节点 | 宣布性报道管理 | 情绪表达与行动指引 |

## 10. 系统架构与现有技术栈复用

### 10.1 现有能力可复用模块

1. `backend/game/story_engine.py`：改造成历史节点推进器
2. `backend/game/event_generator.py`：用于多轮场景生成
3. `backend/database/vector_db.py`：用于历史案例与复盘检索

5. `frontend/src/pages/Game.tsx`：承载交互训练界面
6. `backend/api/routers/tts.py`：承载语音播报与高压语境模拟

### 10.2 新增模块建议

1. `backend/kt/kt_engine.py`：BKT/DKT 更新核心
2. `backend/kt/rubric.py`：能力评分规则与阈值
3. `backend/kt/strategy.py`：下一场景推荐策略器
4. `backend/history/timeline.py`：历史时间轴与事件约束
5. `backend/api/routers/training.py`：训练专用 API
6. `frontend/src/pages/TrainingReport.tsx`：能力雷达与成长报告页

3. 复用向量库保存“错误案例-复盘建议”语义索引

## 12. API 设计（建议）

|---|---|---|
| POST | `/api/v1/training/init` | 初始化训练与能力状态 |
| GET | `/api/v1/training/scenario/next` | 获取下一训练场景 |
| POST | `/api/v1/training/round/submit` | 提交本轮决策并触发评分/KT更新 |

## 13. 输出成果形式

### 13.1 用户侧输出

1. 能力雷达图（K1-K8）
2. 能力成长曲线（按轮次）
3. 关键错误分布与复盘建议
4. 反事实对比报告（本轮选项 A/B 后果差异）

### 13.2 教师侧输出

1. 班级能力分布热力图
2. 高风险能力薄弱人群识别
3. 课程训练效果评估（前测/后测）

## 14. 评测指标（可用于答辩）

### 14.1 模型与算法指标

1. 评分一致性（与专家评分相关性）
2. KT 预测准确率（下一轮表现预测）
3. 校准度（置信度与真实表现一致性）

### 14.2 训练效果指标

1. 能力提升幅度：`ΔK = post - pre`
2. 迁移表现：未见场景下的得分改善
3. 高风险错误率下降（如“未核实发布率”）
4. 达标效率（达到目标能力所需轮次）

## 15. 社会价值与教育价值

1. 提升青年群体媒体素养和风险信息识读能力
2. 通过历史情境训练现实危机沟通能力
3. 在课程中实现“教、练、评、改”闭环
4. 形成可复制的数字化训练模板，支持多校推广

## 16. 合规与伦理设计

1. 历史真实性：关键事实节点有来源标注
2. 训练边界：明确为教学模拟，不替代真实新闻流程
3. 内容安全：对极端表述与不当引导进行拦截
4. 数据安全：训练数据脱敏存储、最小权限访问

## 18. 10分钟答辩演示脚本（建议）

1. 0:00-1:30：问题定义与目标（从“剧情互动”升级到“能力训练”）
2. 1:30-3:00：展示能力图谱与初始雷达图
3. 3:00-5:00：演示场景 S1，提交决策，显示即时评分与证据
4. 5:00-6:30：展示 KT 更新结果与短板识别
5. 6:30-8:00：演示系统自适应推送下一场景
6. 8:00-9:00：展示反事实对比与训练报告
7. 9:00-10:00：总结创新点、社会价值与可推广性

## 19. 创新点总结（可直接用于申报书）

1. 历史情境约束下的高风险沟通训练框架
2. LLM 证据评分与 KT 能力追踪融合
3. 反事实复盘与个性化场景推荐机制
4. 可解释评估与可视化成长报告闭环

## 20. 结论

本项目将“抗战历史教育、风险传播训练、AI个性化学习”三者打通，形成兼具社会价值与技术深度的实践赛作品。其核心不是“让用户改写历史”，而是“让用户在历史情境中训练负责任的沟通与决策能力”，并通过 KT 机制实现可测评、可成长、可复用的智能训练体系。
