# 游戏完整业务逻辑说明

## 一、游戏整体架构

### 1.1 核心组件
- **GameService**: 游戏服务层，处理游戏逻辑
- **GameSession**: 游戏会话（每个thread_id对应一个会话）
- **StoryEngine**: 剧情推进引擎，管理事件和对话
- **EventGenerator**: 事件生成器，负责生成事件和对话
- **GameSessionManager**: 会话管理器（单例模式）

### 1.2 数据存储
- **PostgreSQL**: 存储角色信息、状态值、属性等结构化数据
- **ChromaDB**: 向量数据库，存储历史事件和对话，用于RAG检索

---

## 二、游戏流程详解

### 2.1 游戏初始化阶段

#### 步骤1: 创建游戏会话 (`init_game`)
```
用户请求 → GameService.init_game()
  ↓
创建GameSession（包含thread_id, user_id, character_id, game_mode）
  ↓
初始化组件：
  - DatabaseManager（数据库管理器）
  - VectorDatabase（向量数据库）
  - EventGenerator（事件生成器）
  - StoryEngine（故事引擎）
  ↓
返回 thread_id, user_id, game_mode
```

#### 步骤2: 角色选择与背景去除
```
用户在CharacterSelection页面选择角色图片
  ↓
点击CHOICE按钮
  ↓
调用 removeCharacterBackground API（使用rembg isnet-general-use模型）
  ↓
去除图片背景，生成透明背景PNG
  ↓
保存透明图片URL到sessionStorage
  ↓
跳转到FirstMeetingSelection（初遇场景选择）
```

#### 步骤3: 选择初遇场景 (`initialize_story`)
```
用户在FirstMeetingSelection选择场景（如：学校、图书馆等）
  ↓
调用 initialize_story API
  ↓
GameService.initialize_story(thread_id, character_id, scene_id)
```

## 三、初遇场景事件生成逻辑

### 3.1 初遇事件生成流程 (`get_opening_event`)

```python
# 位置: backend/game/story_engine.py

def get_opening_event(character_id, scene_id='school'):
    # 1. 从SCENES获取场景配置
    scene = SCENES.get(scene_id)
    
    # 2. 获取该场景的opening_events列表
    #    如果场景没有定义opening_events，使用默认事件
    opening_events = scene.get('opening_events', [默认事件])
    
    # 3. 随机选择一个初遇事件
    selected_event = random.choice(opening_events)
    
    # 4. 智能场景映射（重要！）
    #    如果事件ID对应一个场景（如cafeteria事件ID对应cafeteria场景），
    #    则使用事件ID作为场景ID（用于场景图片显示）
    if event_id in SCENES and event_id != 'school':
        event_scene_id = event_id  # 使用事件ID作为场景ID
    
    # 5. 调用EventGenerator生成事件（包含故事背景）
    event = event_generator.generate_event(
        character_id=character_id,
        event_id=selected_event['id'],
        event_context=selected_event['description'],
        previous_contexts=[]
    )
    
    # 6. 设置事件属性
    event['title'] = selected_event['title']
    event['scene'] = event_scene_id  # 使用映射后的场景ID
    event['original_scene'] = scene_id  # 保存原始选择的场景
    
    # 7. 更新当前场景
    self.current_scene = event_scene_id
    
    return event
```

## 四、对话系统

### 4.1 对话轮次生成 (`get_next_dialogue_round`)

```python
# 位置: backend/game/story_engine.py

def get_next_dialogue_round(character_id):
    # 1. 计算当前对话轮次
    dialogue_round = (len(self.dialogue_history) // 2) + 1
    
    # 2. 从向量数据库检索最近的对话（用于连贯性）
    recent_dialogues = vector_db.search_recent_dialogues(
        character_id=character_id,
        event_id=self.current_event['event_id'],
        n_results=5
    )
    
    # 3. 构建对话历史上下文
    previous_dialogues = []
    for item in self.dialogue_history:
        if item['type'] == 'character':
            previous_dialogues.append(f"角色: {item['content']}")
        elif item['type'] == 'player':
            previous_dialogues.append(f"玩家: {item['content']}")
    
    # 4. 生成对话轮次（角色对话 + 玩家选项）
    dialogue_data = event_generator.generate_dialogue_round(
        character_id=character_id,
        story_background=self.current_event['story_background'],
        dialogue_round=dialogue_round,
        previous_dialogues=previous_dialogues
    )
    
    return dialogue_data
```

def generate_character_dialogue(character_id, story_background, dialogue_round, previous_dialogues):
    # 1. 获取角色信息
    character = db_manager.get_character(character_id)
    attributes = db_manager.get_character_attributes(character_id)
    states = db_manager.get_character_states(character_id)
    
    # 2. 构建角色信息字典
    character_info = {
        'personality': character.personality,
        'gender': character.gender,
        'appearance': character.appearance,
        'attributes': attributes
    }
    
    # 3. 构建状态值字典（影响对话风格）
    state_values = {
        'favorability': states.favorability,  # 好感度
        'trust': states.trust,                 # 信任度
        'hostility': states.hostility,         # 敌意值
        'emotion': states.emotion,             # 情绪值
        # ... 其他状态值
    }
    
    # 4. 调用AI生成对话（使用通义千问）
    dialogue = ai_generator.generate_character_dialogue(
        story_background=story_background,
        character_info=character_info,
        state_values=state_values,
        dialogue_round=dialogue_round,
        previous_dialogues=previous_dialogues
    )
    
    return dialogue

### 5.2 处理玩家输入 (`process_input`)

```python
# 位置: backend/api/services/game_service.py

def process_input(thread_id, user_input, option_id=None):
    # 1. 获取会话
    session = game_session_manager.get_session(thread_id)
    
    # 2. 处理玩家选择
    if option_id is not None:
        # 使用选项ID选择选项
        selected_option = session.current_dialogue_round['player_options'][option_id]
        session.story_engine.process_player_choice(character_id, selected_option)
    else:
        # 自由输入（创建中性选项）
        temp_option = {'type': 'neutral', 'text': user_input}
        session.story_engine.process_player_choice(character_id, temp_option)
    
    # 3. 保存对话轮次到向量数据库
    dialogue_round = len(session.story_engine.dialogue_history) // 2
    session.story_engine.save_dialogue_round_to_vector_db(character_id, dialogue_round)
    
    # 4. 判断是否继续对话
    should_continue = session.story_engine.should_continue_dialogue()
    
    if should_continue:
        # 继续当前事件的对话
        dialogue_data = session.story_engine.get_next_dialogue_round(character_id)
        return {
            'character_dialogue': dialogue_data['character_dialogue'],
            'player_options': dialogue_data['player_options'],
            'is_event_finished': False
        }
    else:
        # 当前事件对话结束，进入下一个事件
        session.story_engine.save_event_to_vector_db(character_id)
        
        # 检查游戏是否结束
        if session.story_engine.is_game_finished():
            # 获取结尾事件
            ending_event = session.story_engine.get_ending_event(character_id)
            # ...
        else:
            # 获取下一个中间事件
            next_event = session.story_engine.get_next_event(character_id)
            # ...
```

### 5.3 中间事件生成 (`get_middle_event`)

```python
# 位置: backend/game/story_engine.py

def get_middle_event(character_id, event_number):
    # 1. 从向量数据库获取历史事件（用于推演）
    previous_events = vector_db.search_similar_events(
        character_id=character_id,
        query="剧情发展 事件 场景",
        n_results=5
    )
    
    # 2. 智能场景切换
    possible_scenes = SCENE_DERIVATIONS.get(self.current_scene, [self.current_scene])
    
    # 70%概率切换场景，30%概率保持当前场景
    if random.random() < 0.7 and len(possible_scenes) > 1:
        next_scene = random.choice([s for s in possible_scenes if s != self.current_scene])
    else:
        next_scene = self.current_scene
    
    # 3. 生成事件上下文（使用AI，基于历史事件推演）
    event_context = self._generate_event_context(
        character_id=character_id,
        event_number=event_number,
        previous_events=previous_events['documents'],
        current_scene=next_scene
    )
    
    # 4. 场景切换时生成场景图片并合成
    if next_scene != self.current_scene:
        # 生成场景图片
        scene_image_url = image_service.generate_scene_image(scene_data, next_scene)
        # 合成场景图和人物图
        composite_path = image_service.composite_scene_with_character(...)
    
    # 5. 生成事件
    event = event_generator.generate_event(
        character_id=character_id,
        event_id=f"middle_event_{event_number}",
        event_context=event_context,
        previous_contexts=self.previous_event_contexts
    )
    
    # 6. 更新当前场景
    self.current_scene = next_scene
    
    # 7. 记录事件上下文（避免重复）
    self.previous_event_contexts.append(event_context[:150])
    if len(self.previous_event_contexts) > 10:
        self.previous_event_contexts = self.previous_event_contexts[-10:]
    
    return event
```

### 6.2 结局事件生成 (`get_ending_event`)

```python
# 位置: backend/game/story_engine.py

def get_ending_event(character_id):
    # 1. 获取角色状态值
    states = db_manager.get_character_states(character_id)
    favorability = states.favorability
    trust = states.trust
    hostility = states.hostility
    
    # 2. 根据状态值判定结局类型
    if favorability > 70 and trust > 60 and hostility < 20:
        ending_type = "happy_ending"  # 美好结局
        event_context = "经过一系列事件，你们的关系变得更加亲密..."
    elif favorability < 30 or hostility > 50:
        ending_type = "bad_ending"  # 坏结局
        event_context = "关系走向了不好的方向..."
    elif trust > 50 and favorability > 40:
        ending_type = "neutral_ending"  # 中性结局
        event_context = "你们的关系保持在一个稳定的状态..."
    else:
        ending_type = "open_ending"  # 开放结局
        event_context = "故事还在继续..."
    
    # 3. 生成结局事件
    event = event_generator.generate_event(
        character_id=character_id,
        event_id=f"ending_{ending_type}",
        event_context=event_context
    )
    
    event['title'] = "结局"
    event['ending_type'] = ending_type
    
    return event
```

## 九、完整游戏流程总结

```
1. 用户创建角色
   ↓
2. AI生成角色图片（3张组图，纯白背景）
   ↓
3. 用户选择角色图片
   ↓
4. 使用rembg去除背景（生成透明PNG）
   ↓
5. 用户选择初遇场景
   ↓
6. 初始化故事（initialize_story）
   - 生成初遇事件（随机选择opening_events）
   - 智能场景映射（事件ID → 场景ID）
   - 生成故事背景（使用AI）
   - 生成场景图片
   - 合成场景图和人物图
   - 生成第一轮对话
   ↓
7. 对话循环（至少4轮）
   - 角色说话
   - 玩家选择选项
   - 更新状态值
   - 保存对话到向量数据库
   - 生成下一轮对话
   ↓
8. 事件结束
   - 保存完整事件到向量数据库
   - 判断是否游戏结束
   ↓
9. 如果未结束，生成下一个中间事件
   - 智能场景切换（70%概率）
   - 生成事件上下文（基于历史事件）
   - 生成场景图片并合成
   - 生成对话
   ↓
10. 重复步骤7-9，直到达到最大事件数
   ↓
11. 生成结局事件
   - 根据状态值判定结局类型
   - 生成结局对话
   ↓
12. 游戏结束
```

## 十、关键技术点

### 10.1 RAG（检索增强生成）
- 使用ChromaDB向量数据库存储历史事件和对话
- 生成新事件时检索相似历史事件，确保连续性和避免重复

### 10.2 AI生成
- **故事背景生成**: 使用通义千问，基于历史事件生成
- **角色对话生成**: 考虑角色设定、状态值、对话历史
- **玩家选项生成**: 生成3个选项，包含状态值变化

### 10.3 智能场景系统
- 场景衍生关系（SCENE_DERIVATIONS）
- 智能场景切换（70%概率切换）
- 场景映射（事件ID → 场景ID）

### 10.4 图片生成与合成
- 角色图片：AI生成（纯白背景）→ rembg去除背景（透明PNG）
- 场景图片：AI生成（16:9比例，无人物）
- 合成图片：场景图 + 人物图（透明背景叠加）

- SCENES: 场景定义（名称、描述、opening_events）
- SCENE_DERIVATIONS: 场景衍生关系

## 总结

这是一个基于AI的无限流剧情游戏系统，核心特点：

1. **AI驱动**: 使用通义千问生成故事背景、对话、选项
2. **RAG增强**: 使用向量数据库检索历史事件，确保连续性
3. **智能场景**: 支持场景切换和场景映射
4. **状态系统**: 12个状态值影响对话和结局
5. **图片系统**: AI生成 + 背景去除 + 场景合成
6. **对话系统**: 至少4轮对话，支持连贯性
7. **结局系统**: 基于状态值判定多种结局类型
