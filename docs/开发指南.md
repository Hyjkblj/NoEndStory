## 四、关键开发要点

### 4.1 OpenAI Assistants API 使用

#### 创建 Assistant

```python
from openai import OpenAI

client = OpenAI(api_key=settings.openai_api_key)

assistant = client.beta.assistants.create(
    name="Story Director",
    instructions="""
    你是一个剧情导演 Agent，负责规划剧情走向。
    你需要分析玩家行为，决定下一步剧情节点。
    """,
    model="gpt-4o",
    tools=[
        {
            "type": "function",
            "function": {
                "name": "get_story_state",
                "description": "获取当前剧情状态",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "thread_id": {
                            "type": "string",
                            "description": "线程ID"
                        }
                    },
                    "required": ["thread_id"]
                }
            }
        }
    ]
)

print(f"Assistant ID: {assistant.id}")
```

#### 使用 Thread

```python
# 创建 Thread
thread = client.beta.threads.create()

# 添加消息
message = client.beta.threads.messages.create(
    thread_id=thread.id,
    role="user",
    content="我想去海边看看"
)

# 运行 Assistant
run = client.beta.threads.runs.create(
    thread_id=thread.id,
    assistant_id=assistant.id
)

# 等待完成
while run.status in ["queued", "in_progress"]:
    run = client.beta.threads.runs.retrieve(
        thread_id=thread.id,
        run_id=run.id
    )
    time.sleep(1)

# 获取结果
if run.status == "completed":
    messages = client.beta.threads.messages.list(
        thread_id=thread.id
    )
    print(messages.data[0].content[0].text.value)
```

### 4.2 Function Calling 设计

Function Calling 允许 Assistant 调用外部函数，这对于剧情控制非常重要。

```python
# Function 定义
def get_story_state(thread_id: str) -> dict:
    """获取当前剧情状态"""
    # 从数据库获取状态
    state = db.query_story_state(thread_id)
    return {
        "current_scene": state.scene,
        "emotion_values": state.emotions,
        "flags": state.flags
    }

# 在 Assistant 中处理 Function Calling
if run.required_action:
    tool_calls = run.required_action.submit_tool_outputs.tool_calls
    
    tool_outputs = []
    for tool_call in tool_calls:
        if tool_call.function.name == "get_story_state":
            result = get_story_state(thread_id)
            tool_outputs.append({
                "tool_call_id": tool_call.id,
                "output": json.dumps(result)
            })
    
    # 提交结果
    run = client.beta.threads.runs.submit_tool_outputs(
        thread_id=thread.id,
        run_id=run.id,
        tool_outputs=tool_outputs
    )
```

### 4.4 RAG 记忆系统

```python

embedding = client.embeddings.create(
    model="text-embedding-3-large",
    input="Alice 在海边遇到了 Bob，他们成为了朋友"

# 2. 存储到向量数据库
vector_db.upsert(
    vectors=[embedding.data[0].embedding],
    metadata={
        "thread_id": thread_id,
        "content": "Alice 在海边遇到了 Bob...",
        "scene": "beach"

# 3. 检索相关历史
query_embedding = client.embeddings.create(
    model="text-embedding-3-large",
    input="Alice 和 Bob 的关系如何？"

results = vector_db.query(
    query_vector=query_embedding.data[0].embedding,
    top_k=5,
    filter={"thread_id": thread_id}

# 4. 将检索结果注入 Prompt
context = "\n".join([r.metadata["content"] for r in results])
```

## 五、开发最佳实践

### 5.1 代码组织

- **服务层分离**：业务逻辑放在 `services/`，Agent 逻辑放在 `core/agents/`
- **模型分离**：数据库模型（`models/database.py`）和 API 模型（`models/schemas.py`）分开
- **配置集中**：所有配置统一在 `config.py` 管理

### 5.2 错误处理

```python
from fastapi import HTTPException
import openai

async def generate_story(thread_id: str, user_input: str):
    try:
        result = await openai_service.run_assistant(thread_id, assistant_id)
        return result
    except openai.APIError as e:
        logger.error(f"OpenAI API error: {e}")
        raise HTTPException(
            status_code=502,
            detail=f"AI service error: {str(e)}"
        )
    except Exception as e:
        logger.error(f"Unexpected error: {e}")
        raise HTTPException(
            status_code=500,
            detail="Internal server error"
        )
```

## 八、常见问题

### 8.1 OpenAI API 错误

**问题**：API Key 无效
**解决**：检查 `.env` 文件中的 `OPENAI_API_KEY` 是否正确

**问题**：Rate Limit 错误
**解决**：实现重试机制和请求队列

### 8.2 Thread 管理问题

**问题**：Thread 丢失
**解决**：确保 Thread ID 正确存储在数据库中

**问题**：Thread 过多
**解决**：实现 Thread 清理机制，定期清理过期 Thread

### 8.3 成本控制

**问题**：API 调用成本过高
**解决**：
1. 使用缓存减少重复调用
2. 使用更便宜的模型（如 gpt-4o-mini）处理简单任务
3. 限制用户请求频率

### 8.4 性能优化

**问题**：响应时间过长
**解决**：
1. 异步处理图像生成
2. 使用流式输出
3. 优化数据库查询
4. 使用 Redis 缓存
