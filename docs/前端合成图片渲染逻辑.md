# 前端合成图片渲染逻辑

## 概述
本文档说明前端如何渲染合成图片（场景+人物已合成的图片），以及错误处理机制。

## 1. 状态管理

### 1.1 状态定义
```typescript
const [compositeImageUrl, setCompositeImageUrl] = useState<string | null>(null);
const [shouldUseComposite, setShouldUseComposite] = useState<boolean>(false);
```

- `compositeImageUrl`: 合成图片的URL
- `shouldUseComposite`: 标记是否应该使用合成图片（后端返回了 `composite_image_url`）

### 1.2 状态设置逻辑

当后端返回 `composite_image_url` 时：
```typescript
if (responseData.composite_image_url) {
  setCompositeImageUrl(responseData.composite_image_url);
  setShouldUseComposite(true);  // 标记必须使用合成图片
  setSceneImageUrl(null);       // 清除分别的图片URL
  setCharacterImageUrl(null);
} else {
  setShouldUseComposite(false); // 标记不使用合成图片
}
```

## 2. 渲染逻辑

### 2.1 渲染优先级

1. **如果 `shouldUseComposite === true` 但 `compositeImageUrl === null`**
   - **错误情况**：后端返回了 `composite_image_url`，但URL为空
   - **处理**：报错并显示错误提示，不允许降级到分别显示

2. **如果 `compositeImageUrl` 存在**
   - **必须显示合成图片**：不允许降级到分别显示场景和人物
   - **加载失败处理**：报错并显示错误提示

3. **如果 `compositeImageUrl` 不存在且 `shouldUseComposite === false`**
   - **正常情况**：显示分别的场景和人物图片（叠加显示）

### 2.2 渲染代码

```tsx
{(() => {
  // 错误检查：应该使用合成图片但没有
  if (shouldUseComposite && !compositeImageUrl) {
    console.error('[游戏] 错误：后端返回了composite_image_url，但合成图片URL为空');
    message.error('合成图片加载失败，请刷新页面重试');
    return (
      <div className="scene-placeholder-fallback">
        <Text style={{ color: '#ff4d4f' }}>合成图片加载失败</Text>
      </div>
    );
  }
  
  // 如果有合成图片URL，必须显示合成图片
  if (compositeImageUrl) {
    return (
      <img 
        src={compositeImageUrl} 
        alt="游戏场景" 
        className="composite-scene-image"
        onError={(e) => {
          console.error('[游戏] 错误：合成图片加载失败，URL:', compositeImageUrl);
          message.error('合成图片加载失败，请刷新页面重试');
          // 显示错误占位符
        }}
      />
    );
  }
  
  // 如果没有合成图片，显示分别的场景和人物图片
  return (
    <>
      {/* 场景图片 */}
      {sceneImageUrl && <img src={sceneImageUrl} />}
      {/* 人物图片 */}
      {characterImageUrl && <img src={characterImageUrl} />}
    </>
  );
})()}
```

## 3. 错误处理

### 3.1 错误情况

1. **后端返回了 `composite_image_url` 但URL为空**
   - **检测**：`shouldUseComposite === true && compositeImageUrl === null`
   - **处理**：立即报错，显示错误提示，不允许降级

2. **合成图片加载失败**
   - **检测**：`<img>` 标签的 `onError` 事件
   - **处理**：报错，显示错误提示，显示错误占位符

### 3.2 错误处理原则

- **不允许降级**：如果应该使用合成图片，必须使用合成图片，不能降级到分别显示
- **立即报错**：检测到错误立即报错，不尝试恢复
- **用户提示**：使用 `message.error()` 提示用户刷新页面

## 4. 数据流

```
后端返回响应
    ↓
检查 composite_image_url
    ↓
如果有 → 设置 compositeImageUrl 和 shouldUseComposite = true
    ↓
渲染时检查 shouldUseComposite
    ↓
如果 true 但 compositeImageUrl 为空 → 报错返回
    ↓
如果 compositeImageUrl 存在 → 渲染合成图片
    ↓
如果加载失败 → 报错并显示错误提示
```
