# 前端第一幕逻辑修复说明

## 概述
本文档说明前端第一幕逻辑的修复，确保与后端逻辑同步，包括场景ID处理、场景图片URL构建和过场动画显示。

## 1. 修复内容

### 1.1 场景ID处理
**问题**：前端初始化时使用大场景ID和名称，但后端返回的是小场景ID。

**修复**：
- 优先使用后端返回的小场景ID（`storyData.scene`）
- 过场动画显示小场景名称，而不是大场景名称
- 场景图片URL优先从`smallscenes`目录查找

### 1.2 场景图片URL构建
**问题**：前端构建场景图片URL时，总是使用`scenes`目录，没有优先尝试`smallscenes`目录。

**修复**：
- 优先尝试`smallscenes`目录（小场景图片）
- 备选尝试`scenes`目录（大场景图片）
- URL格式：`/static/images/smallscenes/UNKNOWN_SCENE_{scene_id}_{场景名称}_scene_v1.jpg`

### 1.3 过场动画显示
**问题**：第一幕过场动画显示大场景名称，应该显示小场景名称。

**修复**：
- 使用后端返回的小场景ID获取场景名称
- `setTransitionSceneName(getSceneName(storyData.scene))`，其中`storyData.scene`是小场景ID

## 2. 关键代码修改

### 2.1 FirstMeetingSelection.tsx
**修改**：保存`scene_image_url`到`initialGameData`

```typescript
sessionStorage.setItem('initialGameData', JSON.stringify({
  character_dialogue: storyData.character_dialogue,
  player_options: storyData.player_options,
  composite_image_url: storyData.composite_image_url,
  scene_image_url: storyData.scene_image_url,  // 新增：保存场景图片URL
  scene: storyData.scene  // 小场景ID（后端随机选择的小场景）
}));
```

### 2.2 Game.tsx - 初始化逻辑
**修改**：优先使用后端返回的小场景ID和名称

```typescript

if (gameData.scene) {
```

## 3. 数据流

### 3.1 第一幕初始化流程
```
前端选择大场景（如：school）
    ↓
前端传递大场景ID给后端（initialize_story）
    ↓
后端返回小场景ID（如：playground）和场景图片URL
    ↓
前端保存到initialGameData：
  - scene: "playground"  // 小场景ID
  - scene_image_url: "/static/images/smallscenes/..."  // 小场景图片URL
    ↓
Game组件初始化：
  - 使用小场景ID设置currentScene
  - 使用小场景名称显示过场动画
  - 使用小场景图片URL渲染场景
```

### 3.2 过场动画显示
```
第一幕：
  - 幕数：第一幕
  - 场景名称：小场景名称（如："操场"）
  
后续幕：
  - 幕数：第N幕
  - 场景名称：切换后的小场景名称
```

## 4. 关键保证

### 4.1 场景ID一致性
- ✅ 前端使用后端返回的小场景ID（`storyData.scene`）
- ✅ 不再使用前端选择的大场景ID作为当前场景

### 4.2 场景图片URL正确性
- ✅ 优先使用后端返回的`scene_image_url`（已正确编码）
- ✅ 如果没有，优先尝试`smallscenes`目录
- ✅ 备选尝试`scenes`目录

### 4.3 过场动画正确性
- ✅ 第一幕显示小场景名称，不是大场景名称
- ✅ 场景切换时显示新的小场景名称
