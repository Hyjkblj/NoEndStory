# 事件推进器优化说明

## 优化内容总结

### 1. 中间事件数量调整
- **修改前**：15个中间事件
- **修改后**：7个中间事件
- **文件**：`backend/config.py`
- **配置**：`GAME_CONFIG['max_events'] = 7`

### 2. 对话轮数由AI决定
- **修改前**：固定最少4轮对话
- **修改后**：AI根据对话内容判断是否继续，确保对话有头有尾
- **文件**：`backend/game/story_engine.py`
- **方法**：`should_continue_dialogue(character_id)`
- **逻辑**：
  - 最少2轮对话（保证基本对话）
  - AI分析当前对话历史、故事背景、事件上下文
  - 判断对话是否完整概述了事件或为下一事件做好了铺垫
  - 如果对话已经完整，可以结束；否则继续

### 3. 所有文本都加上名字标识
- **故事背景**：必须包含"player"和角色名（如"player和{角色名}在图书馆相遇..."）
- **角色对话**：必须以"{角色名}:"开头（如"{角色名}: 你好，很高兴认识你"）
- **玩家选项**：必须以"player:"开头（如"player: 你好，我也很高兴认识你"）
- **文件**：
  - `backend/game/ai_generator.py` - 修改了所有生成方法
  - `backend/game/event_generator.py` - 传递角色名参数
  - `backend/game/story_engine.py` - 确保对话格式正确

### 4. 所有生成都基于向量数据库检索
- **故事背景生成**：必须基于向量数据库检索的历史事件
- **事件上下文生成**：必须基于向量数据库检索的历史事件
- **对话生成**：必须基于向量数据库检索的历史对话
- **场景推演**：基于向量数据库检索的历史事件推演场景
- **文件**：
  - `backend/game/story_engine.py` - `_generate_event_context()`、`get_middle_event()`、`get_next_dialogue_round()`
  - `backend/game/event_generator.py` - `generate_story_background()`
  - `backend/game/ai_generator.py` - 所有生成方法都强调必须基于历史事件/对话

### 3. 向量数据库检索强化

#### 事件生成时
```python

all_history_events = vector_db.search_similar_events(

```

### 4. AI Prompt强化

所有AI生成都强调：
1. **必须基于历史**：必须基于历史事件/对话生成，不能脱离历史
2. **明确标识**：必须明确使用"player"和角色名
3. **不能混淆**：玩家和角色的对话必须完全隔离
4. **有头有尾**：对话必须完整概述事件或承上启下

## 使用流程

### 1. 事件生成流程
```
1. 从向量数据库检索历史事件（search_similar_events）
   ↓
2. 推演下一个场景（基于历史事件）
   ↓
3. 生成事件上下文（基于历史事件，包含player和角色名）
   ↓
4. 生成故事背景（基于历史事件，包含player和角色名）
   ↓
5. 开始对话循环
```

### 2. 对话生成流程
```
1. 从向量数据库检索历史对话（search_recent_dialogues + search_similar_events）
   ↓
2. 生成角色对话（基于历史对话，格式："{角色名}: ..."）
   ↓
3. 生成玩家选项（基于历史对话，格式：["player: ...", "player: ...", "player: ..."]）
   ↓
4. 玩家选择后，保存到向量数据库
   ↓
5. AI判断是否继续对话（基于对话完整性）
   ↓
6. 如果继续，回到步骤2；否则结束事件
```

## 注意事项

1. **角色名获取**：所有生成方法都需要获取角色名，如果获取失败，使用默认值"角色"
2. **格式统一**：所有对话都使用英文冒号":"，统一格式为"{名字}: {内容}"
3. **向量数据库**：所有生成都必须基于向量数据库检索，不能一次性生成
4. **历史事件**：如果没有历史事件，会从向量数据库检索；如果检索不到，会明确提示
5. **对话完整性**：AI会判断对话是否完整，确保每个事件都有头有尾
