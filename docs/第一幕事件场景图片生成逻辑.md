# 第一幕事件场景图片生成逻辑

## 概述
本文档说明第一幕（初遇场景）的完整逻辑，包括大场景选择、小场景随机、场景图片生成和图片合成。

## 1. 完整流程

```
前端选择大场景（如：school）
    ↓
前端传递大场景ID给后端（initialize_story）
    ↓
后端接收大场景ID（scene_id）
    ↓
StoryEngine.get_opening_event(character_id, scene_id)
    ↓
从大场景的opening_events中随机选择一个初遇事件
    ↓
获取事件对应的小场景ID（event_sub_scene_id）
    ↓
生成事件（包含故事背景）
    ↓
返回event，其中event['scene'] = event_sub_scene_id（小场景ID）
    ↓
GameService.initialize_story 获取 event_scene = event.get('scene')
    ↓
异步生成合成图片（_generate_composite_image_async）
    ↓
检查是否已有小场景图片（get_latest_scene_image_path(event_scene)）
    ↓
如果没有，生成新的小场景图片（generate_scene_image）
    ↓
小场景图片保存到 smallscenes 目录
    ↓
将小场景图片与角色图片合成（composite_scene_with_character）
    ↓
合成图片保存到 composite 目录
    ↓
返回给前端（composite_image_url）
```

### 2.2 后端接收大场景并随机选择小场景
**文件**: `backend/game/story_engine.py:get_opening_event`

```python
def get_opening_event(self, character_id: int, scene_id: str = 'school', ...):
    # scene_id 是大场景ID（如 'school'）
    major_scene = MAJOR_SCENES.get(scene_id)
    
    # 从大场景的opening_events中随机选择
    selected_event = random.choice(opening_events)
    
    # 获取事件对应的小场景ID
    event_sub_scene_id = selected_event.get('sub_scene')
    if not event_sub_scene_id:
        # 从大场景的小场景中随机选择一个
        sub_scenes = major_scene.get('sub_scenes', [])
        event_sub_scene_id = random.choice(sub_scenes)
    
    # 生成事件
    event = self.event_generator.generate_event(...)
    event['scene'] = event_sub_scene_id  # 使用小场景ID
    event['major_scene'] = scene_id  # 保存大场景ID
    
    return event
```

### 2.3 生成小场景图片
**文件**: `backend/api/services/game_service.py:_generate_composite_image_async`

```python
def _generate_composite_image_async(self, thread_id: str, character_id: int, 
                                    event_scene: str, scene_id: str, ...):
    # event_scene 是小场景ID（如 'playground'）
    # scene_id 是大场景ID（如 'school'）
    
    # 检查是否已有小场景图片
    scene_image_path = image_service.get_latest_scene_image_path(event_scene)
    
    if not scene_image_path:
        # 没有已有图片，生成新的小场景图片
        scene_info = SUB_SCENES.get(event_scene, {})
        scene_data = {
            'scene_id': event_scene,  # 小场景ID
            'scene_name': scene_info.get('name', event_scene),
            'scene_description': scene_info.get('description', '')
        }
        scene_image_url = image_service.generate_scene_image(scene_data, event_scene)
```

### 2.4 小场景图片保存逻辑
**文件**: `backend/api/services/image_service.py:_save_image_to_local`

```python
def _save_image_to_local(self, ..., scene_id: Optional[str] = None, ...):
    if image_type == 'scene':
        # 判断是小场景还是大场景
        from data.scenes import SUB_SCENES, MAJOR_SCENES
        is_small_scene = scene_id and scene_id in SUB_SCENES
        is_major_scene = scene_id and scene_id in MAJOR_SCENES
        
        if is_small_scene:
            # 小场景图片保存到 smallscenes 目录
            save_dir = config.SMALL_SCENE_IMAGE_SAVE_DIR
        elif is_major_scene:
            # 大场景图片保存到 scenes 目录
            save_dir = config.SCENE_IMAGE_SAVE_DIR
        else:
            # 未知场景，默认保存到 smallscenes 目录
            save_dir = config.SMALL_SCENE_IMAGE_SAVE_DIR
```

## 3. 目录结构

### 3.1 场景图片目录
- **`backend/images/scenes/`** - 大场景图片目录
  - 格式：`{major_scene_id}_{场景名称}.jpeg`
  - 示例：`school_学校.jpeg`
  
- **`backend/images/smallscenes/`** - 小场景图片目录（第一幕使用）
  - 格式：`UNKNOWN_SCENE_{scene_id}_{场景名称}_scene_v{version}_{timestamp}.jpg`
  - 示例：`UNKNOWN_SCENE_playground_操场_scene_v5_20260121_232737.jpg`

### 3.2 合成图片目录
- **`backend/images/composite/`** - 合成图片目录（小场景+角色）
  - 格式：`{user_id}_{character_id:04d}_SCENE_{scene_id}_composite_v{version}_{timestamp}.jpg`
  - 示例：`UNKNOWN_0030_SCENE_playground_composite_v1_20260122_015639.jpg`

## 4. 数据流

### 4.1 初始化故事请求
```json
{
  "thread_id": "...",
  "character_id": 30,
  "scene_id": "school",  // 大场景ID
  "character_image_url": "/static/images/characters/..."
}
```

### 4.2 后端处理
1. 接收大场景ID：`scene_id = "school"`
2. 随机选择小场景：`event_sub_scene_id = "playground"`
3. 生成小场景图片：保存到 `smallscenes/UNKNOWN_SCENE_playground_操场_scene_v1_...jpg`
4. 合成图片：保存到 `composite/UNKNOWN_0030_SCENE_playground_composite_v1_...jpg`

### 4.3 返回响应
```json
{
  "event_title": "操场捡球",
  "story_background": "...",
  "scene": "playground",  // 小场景ID
  "character_dialogue": "...",
  "player_options": [...],
  "composite_image_url": "/static/images/composite/UNKNOWN_0030_SCENE_playground_composite_v1_...jpg",
  "scene_image_url": "/static/images/smallscenes/UNKNOWN_SCENE_playground_操场_scene_v1_...jpg"
}
```

## 5. 关键保证

### 5.1 小场景图片生成保证
- ✅ 使用小场景ID（`event_scene`）生成图片
- ✅ 图片保存到 `smallscenes` 目录
- ✅ 通过 `SUB_SCENES` 判断是否为小场景

### 5.2 图片合成保证
- ✅ 使用小场景图片（`smallscenes` 目录）与角色图片合成
- ✅ 合成图片保存到 `composite` 目录
- ✅ 返回正确的 `composite_image_url`

### 5.3 前端渲染保证
- ✅ 优先使用 `composite_image_url`（合成图片）
- ✅ 如果合成图片不存在，使用 `scene_image_url`（小场景图片）和 `character_image_url`（角色图片）分别显示
