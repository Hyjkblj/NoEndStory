# 前端第一幕事件场景图片渲染逻辑

## 概述
本文档说明前端渲染第一幕事件（初遇场景）时，如何获取和渲染场景图片，包括是否使用 `@backend/images/scenes` 目录中的大场景图片。

## 1. 场景图片目录结构

### 1.1 目录说明
- **`backend/images/scenes`** - 大场景图片目录
  - 格式：`{major_scene_id}_{场景名称}.jpeg`
  - 示例：`school_学校.jpeg`, `company_公司.jpeg`, `dailylife_日常生活.jpeg`
  
- **`backend/images/smallscenes`** - 小场景图片目录（优先使用）
  - 格式：`UNKNOWN_SCENE_{scene_id}_{场景名称}_scene_v{version}_{timestamp}.jpg`
  - 示例：`UNKNOWN_SCENE_playground_操场_scene_v5_20260121_232737.jpg`

### 1.2 静态文件服务配置
```python
# backend/api/app.py
app.mount("/static/images/scenes", StaticFiles(directory=SCENE_IMAGE_SAVE_DIR), name="scene_images")
app.mount("/static/images/smallscenes", StaticFiles(directory=SMALL_SCENE_IMAGE_SAVE_DIR), name="small_scene_images")
```

## 2. 后端查找逻辑

### 2.1 查找优先级 (`image_service.py:get_latest_scene_image_path`)

```python
def get_latest_scene_image_path(self, scene_id: str) -> Optional[str]:
    # 1. 优先从 smallscenes 目录查找（小场景图片）
    # 2. 再查找 scenes 目录（大场景图片，作为备选）
    
    search_dirs = [
        SMALL_SCENE_IMAGE_SAVE_DIR,  # 优先
        SCENE_IMAGE_SAVE_DIR          # 备选
    ]
```

**说明**：
- 优先查找小场景图片（`smallscenes` 目录）
- 如果没找到，再查找大场景图片（`scenes` 目录）

### 2.2 返回URL构建 (`game_service.py:initialize_story`)

```python
# 查找最新的场景图片
scene_image_path = image_service.get_latest_scene_image_path(event_scene)

if scene_image_path and os.path.exists(scene_image_path):
    filename = os.path.basename(scene_image_path)
    encoded_filename = quote(filename, safe='')
    
    # 判断图片来自哪个目录
    if scene_image_path.startswith(small_scene_dir):
        scene_image_url = f"/static/images/smallscenes/{encoded_filename}"
    else:
        scene_image_url = f"/static/images/scenes/{encoded_filename}"  # 大场景图片
```

**说明**：
- 如果找到的图片在 `smallscenes` 目录，返回 `/static/images/smallscenes/...`
- 如果找到的图片在 `scenes` 目录，返回 `/static/images/scenes/...`（**大场景图片**）

### 2.3 备选查找逻辑

如果 `get_latest_scene_image_path` 没找到，会尝试查找大场景图片：

```python
# 尝试查找大场景图片（格式：{major_scene_id}_{场景名称}.{ext}）

```

**说明**：
- 如果小场景图片不存在，会尝试查找大场景图片
- 大场景图片格式：`school_学校.jpeg`, `company_公司.jpeg` 等

## 3. 前端渲染逻辑

### 3.1 优先级顺序

1. **后端返回的 `scene_image_url`**（最高优先级）
   ```typescript
   if (storyData.scene_image_url) {
     setSceneImageUrl(storyData.scene_image_url);
   }
   ```
   - 后端已确定图片位置（`smallscenes` 或 `scenes`）
   - 可能返回大场景图片URL：`/static/images/scenes/school_学校.jpeg`

2. **场景配置中的图片**（次优先级）
   ```typescript
   const sceneConfig = SCENE_CONFIGS.find(s => s.id === storyData.scene);
   if (sceneConfig) {
     const sceneUrl = getSceneImageUrl(sceneConfig);
     // 构建URL: /static/images/scenes/{scene_id}_{场景名称}.jpeg
   }
   ```
   - 使用 `buildSceneImageUrl` 构建URL
   - **注意**：这里构建的URL指向 `scenes` 目录（大场景图片）

3. **备选URL列表**（最低优先级）
   ```typescript
   const possibleSceneUrls = [
     `/static/images/scenes/${storyData.scene}_${encodedSceneName}.jpeg`,  // 小场景图片
     `/static/images/scenes/${majorSceneId}_${encodedMajorSceneName}.jpeg`, // 大场景图片（备选）
   ];
   ```
   - 包含大场景图片作为备选

### 3.2 关键代码位置

**初始化故事时** (`Game.tsx:409-450`)

## 4. 是否使用大场景图片

### 4.1 会使用大场景图片的情况

1. **后端返回大场景图片URL**
   - 当 `get_latest_scene_image_path` 在 `smallscenes` 目录没找到时
   - 会在 `scenes` 目录查找大场景图片
   - 返回：`/static/images/scenes/school_学校.jpeg`

2. **前端场景配置构建**
   - `getSceneImageUrl` 构建的URL指向 `scenes` 目录
   - 格式：`/static/images/scenes/{scene_id}_{场景名称}.jpeg`
   - **注意**：这里使用的是 `scenes` 目录，不是 `smallscenes`

3. **备选URL列表**
   - 包含大场景图片作为备选：`/static/images/scenes/{majorSceneId}_{majorSceneName}.jpeg`

### 4.2 不会使用大场景图片的情况

- 后端在 `smallscenes` 目录找到了对应的小场景图片
- 返回的URL是：`/static/images/smallscenes/UNKNOWN_SCENE_playground_操场_scene_v5.jpg`

## 5. 问题分析

### 5.1 当前问题

**前端场景配置构建的URL指向 `scenes` 目录**：
```typescript
// frontend/src/config/scenes.ts
export function buildSceneImageUrl(sceneId: string, sceneName: string, extension: string = '.jpeg'): string {
  const encodedSceneName = encodeURIComponent(sceneName);
  return `/static/images/scenes/${sceneId}_${encodedSceneName}${extension}`;
  // 注意：这里硬编码了 /static/images/scenes/，没有考虑 smallscenes
}
```

**问题**：
- 如果后端返回的 `scene_image_url` 是 `smallscenes` 目录的图片，前端会正确使用
- 但如果后端没有返回，前端使用场景配置构建URL时，**总是构建 `scenes` 目录的URL**
- 这可能导致前端尝试加载大场景图片，而不是小场景图片

### 5.2 实际文件情况

从文件列表看：
- `scenes` 目录：大场景图片（`school_学校.jpeg`, `company_公司.jpeg` 等）
- `smallscenes` 目录：小场景图片（`UNKNOWN_SCENE_playground_操场_scene_v5.jpg` 等）

**结论**：
- 前端**确实会使用** `@backend/images/scenes` 中的大场景图片
- 但这是作为备选方案，优先使用 `smallscenes` 目录中的小场景图片

## 6. 数据流

```
第一幕事件初始化
    ↓
后端查找场景图片
    ↓
优先查找 smallscenes 目录
    ↓
如果找到 → 返回 /static/images/smallscenes/...
    ↓
如果没找到 → 查找 scenes 目录（大场景图片）
    ↓
如果找到 → 返回 /static/images/scenes/{major_scene_id}_{场景名称}.jpeg
    ↓
前端接收 scene_image_url
    ↓
如果后端返回 → 直接使用（可能是大场景图片）
    ↓
如果后端没返回 → 前端构建URL（指向 scenes 目录，大场景图片）
```
