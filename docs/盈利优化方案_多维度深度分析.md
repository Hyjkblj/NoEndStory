## 一、成本结构分析

### 1.2 成本优化空间

| 优化手段 | 预估节省 | 实施难度 | 优先级 |
|---------|---------|---------|--------|
| LLM降级：非关键调用用更便宜的模型（如qwen-turbo） | 30-40% LLM成本 | 低 | P0 |
| TTS缓存复用（已有MD5缓存，扩大命中率） | 20-30% TTS成本 | 低 | P0 |
| 图片按需生成（非每局都生成全部场景图） | 40-50% 图片成本 | 中 | P1 |
| 对话轮次动态调整（免费用户2-3轮，付费用户4-5轮） | 20-30% 整体成本 | 中 | P1 |
| Prompt压缩（减少重复传入的角色属性字段） | 10-15% LLM成本 | 低 | P1 |
| Edge TTS兜底（已有实现，免费用户默认用Edge TTS） | 100% TTS成本（免费用户） | 低 | P0 |

**优化后单局成本可降至 ¥0.50-0.65。**

- 通过赠送星星（签到、分享）培养消费习惯，降低首次付费心理门槛

### 3.3 VIP订阅制

| 等级 | 价格 | 日均成本 | 核心权益 |
|------|------|---------|---------|
| **免费** | ¥0 | - | 每日3局、1存档槽、标清图片、无语音 |
| **月卡** | ¥30/月 | ¥1.0/天 | 每日10局、3存档槽、高清图片、TTS语音、2个专属场景 |
| **年卡** | ¥198/年 | ¥0.54/天 | 无限局数、5存档槽、全部权益、新功能优先体验、专属客服 |
| **终身卡** | ¥498 | - | 年卡永久权益 + 创始人标识 + 参与剧情投票 |

#### 订阅设计原则

1. **锚定效应**：终身卡¥498让年卡¥198显得划算，年卡让月卡¥30显得便宜
2. **沉没成本**：年卡/终身卡用户投入越多，越不舍得离开
3. **社交货币**：创始人标识、VIP专属外观，让用户有炫耀资本
4. **价格敏感测试**：首月半价¥15降低转化门槛

### 4.2 限流与成本控制（P0）

```python
# services/limit_service.py

class LimitService:
    async def check_play_limit(self, user_id: str) -> PlayLimitResult:
        """检查用户是否可以开始新一局"""
        user = await self.get_user(user_id)

        # VIP用户无限局数
        if user.vip_level >= 2:  # 年卡/终身卡
            return PlayLimitResult(allowed=True, reason="vip_unlimited")

        # 月卡用户每日10局
        if user.vip_level == 1:
            daily = await self.get_daily_limit(user_id)
            if daily.free_plays_used < 10:
                return PlayLimitResult(allowed=True, reason="vip_monthly")

        # 免费用户每日3局
        daily = await self.get_daily_limit(user_id)
        if daily.free_plays_used < 3:
            return PlayLimitResult(allowed=True, reason="free")

        # 检查星星余额
        if user.coins >= 30:
            return PlayLimitResult(
                allowed=True,
                reason="coins",
                cost=30,
                message="免费局数已用完，消耗30星星继续？"
            )

        # 余额不足
        return PlayLimitResult(
            allowed=False,
            reason="limit_reached",
            message="今日免费局数已用完，请充值或观看广告获取星星"
        )

    async def deduct_play_cost(self, user_id: str, reason: str):
        """扣除游戏费用"""
        if reason == "free":
            await self.increment_daily_plays(user_id)
        elif reason == "vip_monthly":
            await self.increment_daily_plays(user_id)
        elif reason == "coins":
            await self.deduct_coins(user_id, 30, "play", "开始新一局游戏")
```

```python
# 优化：按需生成 + 缓存复用

class ImageService:
    async def get_scene_image(self, scene_id: str, sub_scene_id: str, user_tier: str):
        """获取场景图片，优先使用缓存"""

        # 1. 检查缓存（相同场景+子场景的图片可复用）
        cached = await self.check_cache(f"scene:{scene_id}:{sub_scene_id}")
        if cached:
            return cached

        # 2. 免费用户：使用预置的低清通用场景图
        if user_tier == "free":
            return self.get_preset_scene_image(scene_id)

        # 3. VIP用户：AI生成专属场景图
        image = await self.generate_scene_image(scene_id, sub_scene_id)
        await self.cache_image(f"scene:{scene_id}:{sub_scene_id}", image)
        return image
```

**优化效果**：免费用户图片成本降至接近0，VIP用户体验不受影响。

```python
class CostMonitor:
    # 日成本上限
    DAILY_COST_LIMIT = 500  # 元/天

    async def check_daily_cost(self):
        """每日成本检查"""
        today_cost = await self.get_today_total_cost()
        if today_cost > self.DAILY_COST_LIMIT:
            # 触发熔断：新用户降级为最低模型，限制图片生成
            await self.activate_cost_saving_mode()
            await self.alert_admin(f"日成本已达¥{today_cost}，触发熔断")

    async def check_user_cost(self, user_id: str):
        """单用户成本检查"""
        user_daily_cost = await self.get_user_today_cost(user_id)
        if user_daily_cost > 10:  # 单用户日消费超过¥10
            # 异常用户，可能在刷量
            await self.flag_user(user_id, "high_cost")
```
