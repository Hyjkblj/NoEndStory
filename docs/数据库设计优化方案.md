## 三、新增表设计

### 3.1 `users` — 用户表

```sql
CREATE TABLE users (
    id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_type         VARCHAR(20) NOT NULL DEFAULT 'guest'
                      CHECK (user_type IN ('guest', 'registered')),

    -- 身份信息（正式用户必填，游客为 NULL）
    username          VARCHAR(50) UNIQUE,
    email             VARCHAR(200) UNIQUE,
    email_verified    BOOLEAN NOT NULL DEFAULT FALSE,
    phone             VARCHAR(20) UNIQUE,
    password_hash     VARCHAR(255),
    avatar_url        VARCHAR(500),

    -- 安全/反滥用
    registration_ip   VARCHAR(45),
    last_ip           VARCHAR(45),
    device_fingerprint_hash VARCHAR(64),
    failed_login_attempts  INT NOT NULL DEFAULT 0,
    locked_until           TIMESTAMP,

    -- 免费体验控制
    free_play_used    BOOLEAN NOT NULL DEFAULT FALSE,
    free_play_used_at TIMESTAMP,

    -- 统计
    total_games_played INT NOT NULL DEFAULT 0,
    last_game_at       TIMESTAMP,

    -- 状态
    status            VARCHAR(20) NOT NULL DEFAULT 'active'
                      CHECK (status IN ('active', 'banned', 'deleted')),
    created_at        TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at        TIMESTAMP NOT NULL DEFAULT NOW(),
    last_login_at     TIMESTAMP
);

-- 索引
CREATE INDEX idx_users_email_active ON users(email) WHERE email IS NOT NULL AND status = 'active';
CREATE INDEX idx_users_registration_ip ON users(registration_ip);
CREATE INDEX idx_users_device_fingerprint ON users(device_fingerprint_hash);
CREATE INDEX idx_users_status ON users(status);
```

#### 用户类型与免费策略

| 用户类型 | 免费次数 | 认证方式 |
|----------|----------|----------|
| `guest` | 1 次（同一 IP 共享 3 次/24h） | 设备指纹 + IP |
| `registered` | 无限制 | 邮箱 + 密码登录 |

#### 游客 → 正式用户升级

```
用户注册时携带游客 token → 后端验证 token 有效性
  → UPDATE users SET user_type='registered', email=?, username=?, password_hash=?
  → 保留已有的 game_plays 记录
```

### 3.2 `user_tokens` — JWT 令牌表

```sql
CREATE TABLE user_tokens (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id     UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    token_hash  VARCHAR(64) NOT NULL,
    token_type  VARCHAR(20) NOT NULL CHECK (token_type IN ('access', 'refresh')),
    expires_at  TIMESTAMP NOT NULL,
    revoked     BOOLEAN NOT NULL DEFAULT FALSE,
    revoked_at  TIMESTAMP,
    ip_address  VARCHAR(45),
    user_agent  TEXT,
    created_at  TIMESTAMP NOT NULL DEFAULT NOW()
);

-- 索引
CREATE UNIQUE INDEX idx_tokens_hash_type ON user_tokens(token_hash, token_type);
CREATE INDEX idx_tokens_user_active ON user_tokens(user_id) WHERE revoked = FALSE;
```

#### 安全机制

| 机制 | 说明 |
|------|------|
| SHA256 哈希存储 | 不存原始 token，防泄露 |
| 吊销支持 | logout / 安全事件可主动吊销 |
| Refresh Token 轮换 | 每次刷新换新 token，旧的失效 |
| 每用户上限 | 最多同时 5 个 active token |

#### JWT 配置（.env）

```bash
JWT_SECRET=your-random-secret-key-at-least-32-chars
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=60
REFRESH_TOKEN_EXPIRE_DAYS=30
```

### 3.3 `game_plays` — 游戏记录表

```sql
CREATE TABLE game_plays (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id         UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    thread_id       VARCHAR(36) NOT NULL,
    character_id    INT NOT NULL,
    game_mode       VARCHAR(20) NOT NULL DEFAULT 'solo',
    status          VARCHAR(20) NOT NULL DEFAULT 'playing'
                    CHECK (status IN ('playing', 'completed', 'abandoned')),
    is_free_play    BOOLEAN NOT NULL DEFAULT FALSE,
    ip_address      VARCHAR(45),
    started_at      TIMESTAMP NOT NULL DEFAULT NOW(),
    ended_at        TIMESTAMP,
    ending_type     VARCHAR(50),
    summary         TEXT,

    -- 自动计算列
    duration_seconds INT GENERATED ALWAYS AS (
        CASE WHEN ended_at IS NOT NULL
             THEN EXTRACT(EPOCH FROM (ended_at - started_at))::INT
             ELSE NULL
        END
    ) STORED
);

-- 索引
CREATE INDEX idx_game_plays_user_status ON game_plays(user_id, status);
CREATE INDEX idx_game_plays_started ON game_plays(started_at);
CREATE INDEX idx_game_plays_char ON game_plays(character_id);
```

### 3.4 `scene_images` — 场景图片池表

```sql
CREATE TABLE scene_images (
    id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    scene_id          VARCHAR(50) NOT NULL,           -- 小场景 ID：classroom
    scene_name        VARCHAR(100) NOT NULL,          -- 中文名：教室
    major_scene_id    VARCHAR(50) NOT NULL,           -- 大场景 ID：school

    -- 图片文件
    image_path        VARCHAR(500) NOT NULL,
    image_url         VARCHAR(500) NOT NULL,
    width             INT,
    height            INT,
    file_size_kb      INT,

    -- 生成记录
    generation_provider VARCHAR(50),                  -- seedream / wanx
    generation_prompt   TEXT,
    generation_cost     DECIMAL(10, 4) DEFAULT 0,

    -- 复用统计
    use_count         INT NOT NULL DEFAULT 0,
    last_used_at      TIMESTAMP,

    -- 状态
    status            VARCHAR(20) NOT NULL DEFAULT 'active'
                      CHECK (status IN ('active', 'disabled', 'deleted')),
    quality_score     FLOAT DEFAULT 5.0
                      CHECK (quality_score BETWEEN 1 AND 10),
    tags              JSON,                           -- ["indoor","daytime","bright"]
    created_at        TIMESTAMP NOT NULL DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_scene_images_lookup ON scene_images(scene_id, status, quality_score DESC);
```

#### 场景图片池策略

```
每个场景维护 5-10 张变体图片
  ├─ MIN_POOL_SIZE = 5   （少于5张时自动触发补充生成）
  └─ MAX_POOL_SIZE = 10  （超过10张时清理低分图片）

随机抽取逻辑：
  SELECT * FROM scene_images
  WHERE scene_id = ? AND status = 'active' AND quality_score >= 3.0
  ORDER BY RANDOM() * quality_score DESC  -- 加权随机
  LIMIT 1

成本估算（以72个小场景为例）：
  全部预热：72 × 5 × $0.10 = $36（一次性投入）
  按需懒加载：$0.10/次（首次用户等待3-5秒）
  推荐策略：预热 TOP 20 高频场景，其余懒加载
```

### 4.3 ChromaDB 安全修复（修复 P0-4）

```python
# 修改前：任何异常 → 自动删除全部数据
except Exception as e:
    self._reset_database()  # ← 灾难

# 修改后：分级处理
except Exception as e:
    logger.critical(f"向量数据库初始化失败: {e}")
    
    if "version" in str(e).lower() or "schema" in str(e).lower():
        # 版本不兼容 → 备份旧数据 → 创建新库
        self._backup_and_reset()
    elif "locked" in str(e).lower():
        # 文件锁定 → 等待重试
        raise RuntimeError("向量数据库文件被占用，请关闭其他进程后重试")
    else:
        # 其他错误 → 拒绝启动，要求人工介入
        raise RuntimeError(
            f"向量数据库无法启动: {e}\n"
            f"请手动检查: {config.VECTOR_DB_PATH}"
        )
```

## 五、安全防御体系

### 5.1 防滥用分层架构

```
Layer 6 (终极)   邮箱验证后才能免费玩        彻底杜绝
Layer 5          人机验证 (Cloudflare Turnstile)  拦截脚本
Layer 4          IP + 设备指纹联合限制        拦截换号攻击
Layer 3          请求频率限制 (Rate Limiting)   拦截并发刷量
Layer 2          Token 消耗监控 + 自动熔断     成本兜底
Layer 1 (基础)   IP 级别免费次数限制           快速止血
```

### 5.2 游客限制策略

| 限制维度 | 阈值 | 时间窗口 |
|----------|------|----------|
| 每 IP 免费次数 | 3 次 | 24 小时 |
| 每 IP 游客创建 | 5 个 | 1 小时 |
| 每设备指纹游客 | 3 个 | 24 小时 |
| 即游客免费游戏 | 1 次/人 | 永久 |

### 5.3 请求频率限制

| 端点 | 游客限制 | 正式用户限制 | 窗口 |
|------|----------|-------------|------|

## 七、API 汇总

### 7.1 认证模块 `POST /api/v1/auth`

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| `POST` | `/auth/guest` | 创建游客身份，返回 JWT | ❌ |
| `POST` | `/auth/register` | 邮箱+密码注册（升级游客） | ❌ |

## 八、关键业务流程

### 8.1 游客访问流程

```
访客 → POST /auth/guest { device_fingerprint }
  → 检查 IP 是否被封禁 (banned_ips)
  → 检查 IP 游客创建频率
  → 创建 user(user_type=guest) → 签发 JWT
  → 返回 access_token + user 信息

游客 → POST /game/init { character_id }
  → 验证 JWT → 获取 user
  → 检查 free_play_used
    ├─ FALSE → 创建 GameSession → 写入 game_plays(is_free_play=true)
    │         → UPDATE users SET free_play_used=TRUE, free_play_used_at=NOW()
    └─ TRUE  → 403 "免费次数已用完，请注册账号"
```

### 8.2 正式用户登录流程

```
用户 → POST /auth/login { email, password }
  → 查找 users WHERE email=? AND user_type='registered'
  → 验证 password_hash (bcrypt)
  → 检查 failed_login_attempts → 超过5次锁定
  → 签发 JWT → 写入 user_tokens
  → 更新 last_login_at
```

### 8.3 场景图片池流程

```
POST /initialize-story → 小场景 = "classroom"

1. SELECT * FROM scene_images WHERE scene_id='classroom' AND status='active'
2. 加权随机抽取一张 (quality_score 高的优先)
3. 如果有 → 返回 image_url，更新 use_count（UPDATE ... SET use_count = use_count + 1）
4. 如果没有 → 异步生成新图片：
   ├─ Seedream API 生成 → 保存文件
   ├─ INSERT INTO scene_images
   └─ 返回 image_url（首次用户等待 3-5s，后续用户复用）
```
