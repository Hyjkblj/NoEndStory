# 性能优化最佳实践 - 减少耗时

## 一、核心原则

1. **异步优先**：非关键路径操作异步处理
2. **缓存优先**：重复数据使用缓存
3. **并行处理**：可并行操作同时执行
4. **快速响应**：先返回核心数据，后台处理其他

## 二、后端优化

### 1. 图片生成优化（最重要）

#### 当前问题
- 图片生成是同步的，阻塞整个请求
- 每次都要重新生成，没有缓存

#### 优化方案

**方案A：异步生成 + 占位符（推荐）**
```python
# 立即返回，图片异步生成
async def initialize_story(...):
    # 1. 快速返回核心数据（对话、选项）
    dialogue_data = session.story_engine.get_next_dialogue_round(character_id)
    
    # 2. 异步生成图片（不阻塞）
    asyncio.create_task(generate_images_async(...))
    
    # 3. 先返回占位符或已有图片
    return {
        'character_dialogue': dialogue_data['character_dialogue'],
        'player_options': dialogue_data['player_options'],
        'composite_image_url': None,  # 或使用占位符
        'image_generating': True  # 标记正在生成
    }
```

**方案B：场景图片缓存（立即可用）**
```python
# 场景图片预生成并缓存
# 在系统启动时或后台任务中预生成常用场景图片
# 使用时直接读取，无需等待
```

**方案C：复用已有图片**
```python
# 检查是否已有相同场景的图片
existing_image = check_existing_scene_image(scene_id)
if existing_image:
    return existing_image  # 直接使用，不生成
```

## 三、前端优化

### 1. 乐观更新（Optimistic Update）

```typescript

const handleOptionSelect = async (optionId: number) => {

    // 4. 回滚或显示错误
```

### 4. 请求去重

```typescript
// 避免重复请求相同数据
const requestCache = new Map();
const getCharacterImages = async (characterId: string) => {
  if (requestCache.has(characterId)) {
    return requestCache.get(characterId);
  }
  const promise = api.get(`/v1/characters/${characterId}/images`);
  requestCache.set(characterId, promise);
  return promise;
};
```

## 五、代码示例

### 后端：异步图片生成

```python
import asyncio
from concurrent.futures import ThreadPoolExecutor

executor = ThreadPoolExecutor(max_workers=2)

def initialize_story(self, thread_id, character_id, scene_id, ...):
    # 1. 快速返回核心数据
    dialogue_data = session.story_engine.get_next_dialogue_round(character_id)
    
    # 2. 异步生成图片（不阻塞）
    future = executor.submit(
        self._generate_images_sync,
        thread_id, character_id, scene_id, ...
    )
    
    # 3. 立即返回
    return {
        'character_dialogue': dialogue_data['character_dialogue'],
        'player_options': dialogue_data['player_options'],
        'composite_image_url': None,
        'image_task_id': future.task_id  # 用于后续查询
    }
```

### 前端：乐观更新

