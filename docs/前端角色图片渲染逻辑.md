# 前端角色图片渲染逻辑

## 概述
本文档详细说明前端如何获取、管理和渲染角色图片的完整流程。

## 1. 状态管理

### 1.1 状态定义
在 `Game.tsx` 中定义角色图片状态：
```typescript
const [characterImageUrl, setCharacterImageUrl] = useState<string | null>(null);
```

### 1.2 图片优先级
角色图片按以下优先级获取和显示：
1. **透明背景图片** (`transparentImageUrl`) - 最高优先级
2. **原图** (`originalImageUrl`) - 次优先级
3. **普通图片** (`imageUrl`) - 最低优先级

## 3. 图片获取流程

### 3.1 游戏初始化流程 (`Game.tsx:335-470`)

#### 步骤1: 从 sessionStorage 获取
```typescript
const characterDataStr = sessionStorage.getItem('characterData');
if (characterDataStr) {
  const characterData = JSON.parse(characterDataStr);
  // 按优先级检查

    characterImageUrlForInit = characterData.transparentImageUrl;
  } else if (characterData.originalImageUrl) {
    characterImageUrlForInit = characterData.originalImageUrl;
  } else if (characterData.imageUrl) {
    characterImageUrlForInit = characterData.imageUrl;
  }

```

#### 步骤2: 传递给后端初始化故事
```typescript
const storyResponse = await initializeStory(
  newThreadId, 
  charId, 
  undefined, 
  characterImageUrlForInit  // 传递给后端
);
```

#### 步骤3: 初始化后设置角色图片状态
```typescript
// 如果有场景图片（无合成图片）
if (storyData.scene_image_url) {
  // 再次从 sessionStorage 获取并设置状态
  const characterDataStrForImage = sessionStorage.getItem('characterData');
  if (characterDataStrForImage) {
    const characterData = JSON.parse(characterDataStrForImage);
    if (characterData.transparentImageUrl) {
      setCharacterImageUrl(characterData.transparentImageUrl);
    } else if (characterData.originalImageUrl) {
      setCharacterImageUrl(characterData.originalImageUrl);
    } else if (characterData.imageUrl) {
      setCharacterImageUrl(characterData.imageUrl);
    }
  }
  
  // 如果 sessionStorage 中没有，从 API 获取
  if (!characterImageUrl && charId) {
    getCharacterImages(charId)
      .then((imagesResponse) => {
        // 注意：响应拦截器已经提取了 data 字段
        // imagesResponse 格式: { code: 200, message: 'success', data: { images: [...] } }
        // 兼容处理：优先使用 data.images，如果没有则使用 images（兼容旧格式）
        const images = imagesResponse?.data?.images || imagesResponse?.images;
        if (images && Array.isArray(images) && images.length > 0) {
          setCharacterImageUrl(images[0]);  // 使用第一张图片
        }
      })
      .catch((error) => {
        console.warn('获取角色图片失败（不影响游戏）:', error);
      });
  }
}
```

## 6. 关键逻辑点

### 6.1 图片获取策略
1. **优先使用本地缓存**：从 `sessionStorage` 获取，避免不必要的 API 调用
2. **降级策略**：透明背景 → 原图 → 普通图片
3. **异步获取**：API 调用不阻塞游戏流程，失败时只记录警告

### 6.2 错误处理
- API 调用失败不影响游戏进行
- 图片加载失败时隐藏图片元素
- 所有错误都有日志记录

### 6.3 性能优化
- 使用 `sessionStorage` 缓存，减少 API 调用
- 异步加载，不阻塞 UI
- 60秒超时时间，适应图片处理需求

## 8. 常见问题

### 8.1 角色图片不显示
**可能原因**：
1. `sessionStorage` 中没有 `characterData`
2. `characterData` 中没有有效的图片 URL
3. API 调用失败且没有降级处理
4. 图片 URL 路径错误或文件不存在

**解决方案**：
- 检查浏览器控制台日志
- 验证 `sessionStorage.getItem('characterData')` 的内容
- 检查网络请求是否成功
- 验证图片 URL 是否可访问

### 8.2 图片加载失败
**处理方式**：
- `onError` 回调会隐藏图片元素
- 记录错误日志到控制台
- 不影响游戏其他功能

## 9. 响应拦截器处理

### 9.1 响应拦截器 (`api.ts:28-30`)
```typescript
api.interceptors.response.use(
  (response) => {
    return response.data;  // 直接返回 response.data
  },
  // ...
);
```

### 9.2 数据格式转换
**后端返回**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "images": ["/static/images/characters/...", ...]
  }
}
```

**拦截器处理后**（前端接收）：
```typescript
{
  code: 200,
  message: 'success',
  data: {
    images: ["/static/images/characters/...", ...]
  }
}
```

### 9.3 兼容处理
由于响应拦截器已经提取了 `response.data`，代码中使用兼容方式访问：
```typescript
const images = imagesResponse?.data?.images || imagesResponse?.images;
```
- `imagesResponse?.data?.images` - 标准格式（推荐）
- `imagesResponse?.images` - 兼容旧格式（如果后端直接返回 images）
