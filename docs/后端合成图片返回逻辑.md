## 概述
本文档说明后端如何生成、验证和返回合成图片URL，确保返回的URL一定是正确的。

## 2. URL验证机制

### 2.1 验证函数 (`_validate_composite_image_url`)
```python
def _validate_composite_image_url(self, filepath: str) -> Optional[str]:
    """验证合成图片文件并返回正确的URL
    
    Args:
        filepath: 合成图片的本地文件路径
        
    Returns:
        如果文件存在，返回URL编码后的静态文件URL；否则返回None
    """
    # 1. 验证文件存在
    if not os.path.exists(filepath):
        return None
    
    # 2. URL编码文件名
    filename = os.path.basename(filepath)
    encoded_filename = quote(filename, safe='')
    composite_url = f"/static/images/composite/{encoded_filename}"
    
    # 3. 双重验证文件存在
    if os.path.exists(filepath):
        return composite_url
    else:
        return None
```

### 2.2 验证时机

1. **生成合成图片后** (`_generate_composite_image_async`)

2. **查找已生成图片时** (`process_input`)

## 3. URL构建规则

### 3.1 文件命名规则
```
{safe_user_id}_{character_id:04d}_SCENE_{safe_scene_id}_composite_v1_{timestamp}.jpg
```

示例：
```
UNKNOWN_0030_SCENE_playground_composite_v1_20260122_015639.jpg
```

### 3.2 URL构建
```python
# 1. 获取文件名
filename = os.path.basename(composite_path)

# 2. URL编码文件名（处理中文和特殊字符）
from urllib.parse import quote
encoded_filename = quote(filename, safe='')

# 3. 构建静态文件URL
composite_url = f"/static/images/composite/{encoded_filename}"
```

### 3.3 静态文件服务配置
```python
# backend/api/app.py
app.mount("/static/images/composite", StaticFiles(directory=composite_dir), name="composite_images")
```

## 4. 错误处理

### 4.1 文件不存在
- **检测**：`os.path.exists(filepath)` 返回 `False`
- **处理**：返回 `None`，不返回错误的URL
- **日志**：记录错误日志

### 4.2 URL格式错误
- **检测**：URL不以 `/static/images/composite/` 开头
- **处理**：如果是文件路径，转换为URL；否则返回 `None`

### 4.3 文件路径转换
- **检测**：`composite_image_url` 是文件路径而非URL
- **处理**：调用 `_validate_composite_image_url` 转换为URL

## 7. 保证机制

### 7.1 三重验证
1. **生成后验证**：生成合成图片后立即验证
2. **查找时验证**：查找已生成图片时验证
3. **返回前验证**：返回响应前再次验证

### 7.2 URL编码
- 所有文件名都进行URL编码
- 确保中文和特殊字符能正确访问

### 7.3 文件存在性检查
- 每次返回URL前都检查文件是否存在
- 如果文件不存在，返回 `None` 而不是错误的URL
