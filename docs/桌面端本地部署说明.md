# No End Story 桌面端本地部署说明

项目已支持打包为**桌面端应用**（Windows / macOS / Linux），本地部署后可在本机运行，无需浏览器。

---

## 一、架构说明

- **前端**：React + Vite 构建为静态资源，由 **Electron** 加载并作为桌面窗口运行。
- **后端**：Python FastAPI 服务，需在**本机单独运行**，默认 `http://localhost:8000`。
- **桌面端与后端**：打包后的桌面应用会主动请求 `http://localhost:8000` 的 API 与静态资源（/api、/static、/health）。

因此：**使用桌面端时，必须先启动后端**，再启动或使用已打包的桌面应用。

---

## 二、环境要求

- **Node.js**：建议 18+（用于构建前端与 Electron）
- **Python**：建议 3.10+（用于运行后端）
- **系统**：Windows 10/11、macOS、或常见 Linux 发行版

---

## 三、开发模式（边改边看）

1. **启动后端**（在项目根目录或 backend 目录）：
   ```bash
   cd backend
   python run_api.py
   ```
   或：
   ```bash
   python backend/run_api.py
   ```
   确保控制台无报错，且访问 http://localhost:8000/health 返回正常。

2. **启动桌面端开发模式**（新开一个终端）：
   ```bash
   cd frontend
   npm install
   npm run electron:dev
   ```
   - 会自动启动 Vite 开发服务器（端口 3000）并打开 Electron 窗口。
   - 窗口内加载的是 http://localhost:3000，Vite 会把 /api、/static、/health 代理到 8000。

3. 使用方式与浏览器访问 http://localhost:3000 一致，只是跑在桌面窗口里。

---

## 四、打包为可安装的桌面应用

1. **先构建前端并打包 Electron**：
   ```bash
   cd frontend
   npm install
   npm run electron:build
   ```
   或只生成未打包目录（便于测试）：
   ```bash
   npm run electron:pack
   ```

2. **输出位置**（按 `frontend/package.json` 的 build 配置）：
   - 安装包/可执行文件在 **`frontend/release/`** 下。
   - **Windows**：会生成 `.exe` 安装程序（NSIS）。
   - **macOS**：会生成 `.dmg` 安装包。
   - **Linux**：会生成 `AppImage` 等。

3. **图标**（可选）：
   - 若需自定义图标，将图标放到 `frontend/build/` 下：
     - Windows：`icon.ico`
     - macOS：`icon.icns`
     - Linux：`icon.png`
   - 若不存在，Electron 会使用默认图标。

---

## 五、使用打包后的桌面端

1. **先启动后端**（与开发时相同）：
   ```bash
   cd backend
   python run_api.py
   ```

2. **再运行桌面端**：
   - **Windows**：从 `frontend/release/` 安装生成的安装包，或运行未打包目录中的可执行文件。
   - **macOS**：打开生成的 `.dmg`，将应用拖入“应用程序”。
   - **Linux**：运行生成的 AppImage 等。

3. 桌面端启动后会自动请求 `http://localhost:8000` 的 API 和 /health；若后端未启动，会提示无法连接服务器。

---

## 六、注意事项

| 项目 | 说明 |
|------|------|
| 后端必须先启动 | 桌面端依赖本机 `http://localhost:8000`，否则无法进入游戏。 |
| 端口占用 | 确保 8000 端口未被其他程序占用。 |
| 生产环境 API | 打包后前端已配置为在 Electron 内使用 `http://localhost:8000`（API、health、静态资源）。 |
| 安全与网络 | 当前仅连接本机后端，不对外网开放；若需部署到其他机器，需自行改后端地址或做配置/安装说明。 |

---

## 七、可选：一键启动“后端 + 桌面端”

若希望“双击一个脚本就同时起后端和桌面端”，可以：

- **Windows**：写一个 `.bat` 或 PowerShell 脚本，先启动 `python backend/run_api.py`（或 `run_api.py` 所在路径），等待几秒后再启动 `frontend/release/` 下的可执行文件。
- **macOS / Linux**：写一个 shell 脚本，先后台启动 `python backend/run_api.py`，再执行 Electron 打包出的可执行文件。

如需，我可以按你当前目录结构写一份示例脚本（Windows + 一份 shell）。

---

**总结**：本地部署桌面端 = 本机运行后端 + 用现有 Electron 配置打包/运行前端；当前项目已支持该方式，按上述步骤即可实现。
