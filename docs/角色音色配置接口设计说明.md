# 角色音色配置接口设计与服务位置说明

## 📋 概述

角色音色配置用于存储和管理每个角色的语音合成设置，包括预设音色选择、自定义参数等。本文档说明相关的接口设计、服务位置和数据流。

---

## 🏗️ 架构层次

```
前端页面层
    ↓ (API调用)
前端服务层 (api.ts)
    ↓ (HTTP请求)
后端API路由层 (tts.py)
    ↓ (服务调用)
后端服务层 (tts_service.py)
    ↓ (数据访问)
数据存储层 (内存缓存 / 数据库 - TODO)
```

### 2.3 预设音色ID解析方法

**方法定义**：
```python
def _resolve_preset_voice_id(self, preset_id: Optional[str]) -> Optional[str]:
    """将预设音色 ID（如 female_001）解析为火山引擎音色名"""
    if not preset_id:
        return None
    try:
        from data.preset_voices import get_preset_voice
        voice = get_preset_voice(preset_id)
        if voice and voice.get('voice_id'):
            return voice['voice_id']
    except Exception:
        pass
    return preset_id  # 已是火山引擎音色名则原样返回
```

**代码位置**：`backend/api/services/tts_service.py:388-399`

**功能说明**：
- 将前端使用的预设音色ID（如 `female_002`）转换为后端TTS服务使用的音色ID（如 `BV002_streaming`）
- 通过 `data/preset_voices.py` 中的 `get_preset_voice` 函数查询映射关系

### 2.4 在语音合成中使用音色配置

**使用位置**：`backend/api/services/tts_service.py:228-236`

```python
def generate_speech(self, ...):
    # ...
    if override_voice_id:
        # 试听时使用 override_voice_id
        voice_id = self._resolve_preset_voice_id(override_voice_id) or override_voice_id
    else:
        # 从角色配置获取音色
        voice_config = self.get_character_voice_config(character_id)
        raw_voice_id = voice_config.get('preset_voice_id') or self._get_default_voice(character_id)
        voice_id = self._resolve_preset_voice_id(raw_voice_id) if raw_voice_id else raw_voice_id
```

**功能说明**：
- 在生成语音时，从角色音色配置中获取预设音色ID
- 将预设音色ID解析为实际的TTS服务音色ID
- 如果没有配置，使用默认音色

## 🎨 四、前端页面层

### 4.1 角色选择页面（保存音色配置）

**文件位置**：`frontend/src/pages/CharacterSelection.tsx`

**代码位置**：`frontend/src/pages/CharacterSelection.tsx:289-314`

**功能流程**：
1. 用户在音色选择界面选择音色
2. 点击确认后，调用 `setVoiceConfig` 保存到后端
3. 同时保存到前端 `sessionStorage`（`gameStorage.setCharacterData`）

**关键代码**：
```typescript
// 保存音色配置到角色数据和后端
if (selectedVoiceId) {
  try {
    // 保存到后端
    await setVoiceConfig({
      character_id: Number(characterId),
      voice_type: 'preset',
      preset_voice_id: selectedVoiceId,
    });
    
    const characterData = gameStorage.getCharacterData();
    if (characterData) {
      const selectedVoice = presetVoices.find(v => v.id === selectedVoiceId);
      characterData.voiceConfig = {
        voice_type: 'preset',
        preset_voice_id: selectedVoiceId,
        voice_name: selectedVoice?.name || '未知音色',
        voice_description: selectedVoice?.description || '',
        voice_id: selectedVoice?.voice_id || ''
      };
      gameStorage.setCharacterData(characterData);
      console.log('[角色选择] 已保存音色配置:', characterData.voiceConfig);
    }
  } catch (e) {
    console.warn('[角色选择] 保存音色配置失败:', e);
  }
}
```

**数据格式**（前端存储）：
```typescript
interface VoiceConfig {
  voice_type: 'preset';
  preset_voice_id: string;  // 如 'female_002'
  voice_name: string;        // 如 '温柔女声'
  voice_description: string;  // 如 '温柔甜美女声，适合故事叙述、温馨对话'
  voice_id: string;          // 如 'BV002_streaming'
}
```

## ⚠️ 七、注意事项与待优化项

### 7.1 当前实现限制

1. **内存缓存**：音色配置目前存储在内存中（`tts_service.voice_configs`），服务重启后会丢失
2. **数据库持久化**：代码中有 TODO 注释，计划实现数据库持久化
3. **前端获取接口**：前端未实现 `getVoiceConfig` 函数，但后端接口已提供

### 7.2 建议优化

1. **实现数据库持久化**：
   - 创建 `CharacterVoice` 表（参考 `docs/玩家自定义音色实现方案.md`）
   - 在 `get_character_voice_config()` 中实现数据库查询
   - 在 `set_voice_config()` 中实现数据库保存

2. **前端实现获取接口**：
   - 在 `api.ts` 中添加 `getVoiceConfig()` 函数
   - 在需要时从后端获取最新配置

3. **配置同步**：
   - 确保前端 `sessionStorage` 和后端配置保持一致
   - 游戏初始化时可以从后端获取最新配置

## 🎯 十、总结

角色音色配置的完整流程：

1. **设置阶段**：用户在角色选择页面选择音色 → 调用后端接口保存 → 同时保存到前端存储
2. **使用阶段**：游戏初始化时读取配置 → TTS 服务根据配置生成语音
3. **数据存储**：当前为内存缓存 + 前端 sessionStorage，计划实现数据库持久化

**关键接口**：
- `POST /api/v1/tts/voice/config` - 设置音色配置
- `GET /api/v1/tts/voice/config/{character_id}` - 获取音色配置

**关键服务**：
- `TTSService.get_character_voice_config()` - 获取角色音色配置
- `TTSService._resolve_preset_voice_id()` - 解析预设音色ID
