# 动态剧本系统设计方案

> 规则引擎控制 + AI 创作分离架构
> 更新日期: 2026-06-17

## 一、系统概述

### 1.1 设计理念

```
传统方案：AI 同时负责「创作」和「遵守约束」→ 两件事都做不好
新方案：  AI 只负责「创作」，规则引擎负责「控制」→ 各司其职
```

### 1.2 核心架构

```

### 1.3 职责分离

| 组件 | 职责 | 不负责 |
|------|------|--------|
| **剧本管理器** | 生成事件规划、管理草案队列、剪枝 | 不生成台词 |
| **规则引擎** | 情绪计算、剧情推进、结局判定 | 不创作内容 |
| **AI 创作层** | 生成台词、选项文本 | 不做决策 |
| **游戏状态管理** | 存储状态、持久化、记忆管理 | 不做计算 |

### 2.2 草案结构

```python
@dataclass
class EventDraft:
    """事件草案"""
    draft_id: str
    title: str                          # 事件标题
    scene: str                          # 场景 ID
    narrative_beat: str                 # 叙事阶段：opening/rising/climax/resolution
    brief_description: str              # 一句话剧情描述（50-100字）
    key_beats: list[KeyBeat]            # 关键节拍列表（近程事件有，远程事件无）
    emotion_direction: dict             # 情感走向（如 {"favorability": "up", "trust": "up"}）
    estimated_rounds: int               # 预估轮次数
    detail_level: str                   # "full"（近程）或 "brief"（远程）

@dataclass
class KeyBeat:
    """关键节拍：剧情必须推进到的节点"""
    beat_index: int
    trigger_round: int                  # 触发轮次（如 "第 2 轮"）
    narrative_content: str              # 必须传达的剧情信息（一句话）
    emotion_target_range: dict          # 目标情绪范围（如 {"favorability": [40, 70]}）
    option_directions: list[str]        # 选项方向（如 ["积极回应", "犹豫沉默", "拒绝"]）
    scene_mood: str                     # 场景氛围（如 "温馨"、"紧张"、"尴尬"）

@dataclass
class EventSummary:
    """已完成事件的摘要"""
    event_id: str
    title: str
    scene: str
    rounds_played: int
    key_events: str                     # 关键情节摘要（100字）
    emotion_changes: dict               # 情绪变化（如 {"favorability": +25}）
    player_choices_summary: str         # 玩家选择倾向摘要
```

### 2.3 事件 0 的特殊处理（开场事件）

事件 0 的轮次 1 使用**场景+事件池随机选取**，不依赖 AI 生成：

```python
def initialize_opening_event(
    self,
    scene_id: str,
    character_id: int,
) -> tuple[EventDraft, OpeningEvent]:
    """初始化开场事件（事件 0）

    轮次 1：从场景事件池随机选取（0 延迟，质量稳定）
    轮次 2-N：AI 根据轮次 1 的选择继续创作
    """

    # 1. 从场景事件池随机选取开场事件
    opening_event = self._select_opening_from_pool(scene_id)
    # 例如：「借笔」「成为同桌」「图书馆偶遇」「食堂拼桌」

    # 2. 构建事件 0 的 KeyBeat
    event_0_key_beats = [
        KeyBeat(
            beat_index=0,
            trigger_round=0,
            narrative_content=opening_event.description,  # 预设内容
            emotion_target_range={"favorability": [45, 60]},
            option_directions=opening_event.option_directions,  # 预设选项方向
            scene_mood=opening_event.mood,
        ),
        # 轮次 2-N 的 KeyBeat 由 AI 在轮次 1 结束后生成
    ]

    # 3. 创建事件 0 草案
    event_0 = EventDraft(
        draft_id="event_0",
        title=opening_event.title,
        scene=opening_event.scene,
        narrative_beat="opening",
        brief_description=opening_event.description,
        key_beats=event_0_key_beats,
        emotion_direction={"favorability": "up", "trust": "up"},
        estimated_rounds=3,
        detail_level="full",
        opening_event=opening_event,  # 标记为开场事件
    )

    return event_0, opening_event


def _select_opening_from_pool(self, scene_id: str) -> OpeningEvent:
    """从场景事件池随机选取开场事件

    复用现有系统：data/scenes.py 中每个场景的 opening_events
    """
    from data.scenes import MAJOR_SCENES

    major_scene = MAJOR_SCENES.get(scene_id, {})
    opening_events = major_scene.get('opening_events', [])

    if not opening_events:
        # 降级：使用默认开场
        return OpeningEvent(
            title="初遇",
            description="你在校园里偶然遇到了 TA。",
            scene="classroom",
            mood="自然",
            option_directions=["主动打招呼", "默默观察", "假装没看见"],
        )

    # 随机选取一个
    selected = random.choice(opening_events)

    return OpeningEvent(
        title=selected.get('title', '初遇'),
        description=selected.get('description', ''),
        scene=selected.get('scene', scene_id),
        mood=selected.get('mood', '自然'),
        option_directions=selected.get('option_directions', [
            "积极回应", "保持距离", "转移话题"
        ]),
    )
```

def advance_script(self, completed_event: EventSummary) -> None:
    """事件结束后推进剧本"""

    # 1. 存储已完成事件摘要
    self.completed_events.append(completed_event)

    # 2. 移除已完成的草案
    self.draft_queue.pop(0)

    # 3. 升级下一个草案为完整版
    if self.draft_queue:
        next_draft = self.draft_queue[0]
        if next_draft.detail_level == "brief":
            upgraded = upgrade_to_full_draft(
                draft=next_draft,
                completed_events=self.completed_events,
                current_emotion=get_current_emotion(),
                anchor=self.narrative_anchor
            )
            self.draft_queue[0] = upgraded

    # 4. 追加新的简略草案（保持 3-4 个领先）
    while len(self.draft_queue) < 4:
        new_draft = generate_brief_event_draft(
            anchor=self.narrative_anchor,
            beat_index=len(self.completed_events) + len(self.draft_queue),
            detail_level="brief"
        )
        self.draft_queue.append(new_draft)

    # 5. 剪枝：删除与当前走向不符的草案
    self._prune_inconsistent_drafts()
```

## 三、规则引擎（RuleEngine）

### 3.1 情绪计算

```python
class RuleEngine:
    """规则引擎：负责所有计算和决策"""

    def calculate_emotion_change(
        self,
        current: EmotionState,
        option_state_changes: dict,      # 选项预设的变化量
        narrative_beat: str,             # 当前叙事阶段
        beat_emotion_target: dict,       # 节拍的情绪目标范围
    ) -> EmotionState:
        """计算情绪变化（纯规则，不依赖 AI）"""

        # 1. 叙事阶段修正系数
        stage_multiplier = {
            'opening': 0.6,
            'rising': 1.0,
            'climax': 1.5,
            'resolution': 0.8,
        }
        multiplier = stage_multiplier.get(narrative_beat, 1.0)

        # 2. 应用选项的 state_changes
        new_emotion = current.copy()
        for dim, delta in option_state_changes.items():
            current_val = getattr(new_emotion, dim, 50)
            new_val = current_val + delta * multiplier
            setattr(new_emotion, dim, clamp(new_val, 0, 100))

        # 3. 情绪衰减（防止极端值长期不变）
        new_emotion = self._apply_decay(new_emotion)

        # 4. 节拍目标软约束（如果情绪偏离目标范围，给予轻微修正）
        new_emotion = self._apply_beat_soft_constraint(new_emotion, beat_emotion_target)

        return new_emotion

    def _apply_decay(self, emotion: EmotionState) -> EmotionState:
        """情绪衰减：极端值向中间回归"""
        decay_rate = 0.02  # 每轮衰减 2%

        for dim in ['stress', 'anxiety', 'hostility']:
            val = getattr(emotion, dim)
            if val > 50:
                setattr(emotion, dim, val - (val - 50) * decay_rate)

        for dim in ['happiness', 'sadness']:
            val = getattr(emotion, dim)
            if abs(val - 50) > 20:
                setattr(emotion, dim, val + (50 - val) * decay_rate)

        return emotion

    def _apply_beat_soft_constraint(
        self,
        emotion: EmotionState,
        target_range: dict
    ) -> EmotionState:
        """节拍目标软约束：如果情绪偏离目标，给予轻微修正"""
        correction_strength = 0.1  # 修正强度 10%

        for dim, (low, high) in target_range.items():
            val = getattr(emotion, dim)
            if val < low:
                setattr(emotion, dim, val + (low - val) * correction_strength)
            elif val > high:
                setattr(emotion, dim, val - (val - high) * correction_strength)

        return emotion
```

### 3.2 剧情推进判定

```python
def should_advance_event(
    self,
    current_round: int,
    dialogue_history: list,
    current_beat: KeyBeat,
    emotion: EmotionState,
) -> tuple[bool, str]:
    """判定当前事件是否应该结束"""

    # 1. 所有关键节拍已完成
    all_beats_done = all(
        beat.trigger_round <= current_round
        for beat in current_beat.parent_event.key_beats
    )

    # 2. 最少轮次保护
    min_rounds = current_beat.parent_event.estimated_rounds - 1
    if current_round < min_rounds:
        return False, "未达到最少轮次"

    # 3. 最多轮次硬上限
    max_rounds = current_beat.parent_event.estimated_rounds + 2
    if current_round >= max_rounds:
        return True, "达到最多轮次"

    # 4. 关键节拍全部完成 + 情绪稳定 → 结束
    if all_beats_done:
        emotion_stability = self._calculate_stability(emotion)
        if emotion_stability > 0.7:  # 情绪变化趋于稳定
            return True, "关键节拍完成且情绪稳定"

    # 5. 渐进式结束概率（超过预估轮次后）
    if current_round > current_beat.parent_event.estimated_rounds:
        excess = current_round - current_beat.parent_event.estimated_rounds
        prob = min(0.6, excess * 0.2)
        if random.random() < prob:
            return True, "渐进式结束"

    return False, "继续"
```

### 3.3 结局触发判定

```python
def should_trigger_ending(
    self,
    total_rounds: int,
    total_events: int,
    emotion: EmotionState,
    narrative_beat: str,
    anchor_progress: float,  # 锚点完成度 0-1
) -> tuple[bool, str, str]:
    """判定是否触发结局

    Returns:
        (是否触发, 触发原因, 结局类型)
    """

    # ===== 硬上限 =====
    if total_rounds >= 25:
        return True, "轮次硬上限", self._classify_ending(emotion)

    if total_events >= 7:
        return True, "事件硬上限", self._classify_ending(emotion)

    # ===== 情绪极化触发 =====
    if total_rounds >= 8:
        # 美好结局提前触发
        if (emotion.favorability >= 85 and
            emotion.trust >= 75 and
            emotion.happiness >= 70):
            return True, "情绪极化-美好", "happy_ending"

        # 悲剧结局提前触发
        if emotion.hostility >= 70:
            return True, "情绪极化-敌意", "bad_ending"

        if emotion.favorability <= 15 and emotion.trust <= 20:
            return True, "情绪极化-崩塌", "bad_ending"

    # ===== 锚点完成度 + 轮次联合判定 =====
    if total_rounds >= 12 and anchor_progress >= 0.75:
        return True, "锚点基本完成", self._classify_ending(emotion)

    # ===== 叙事阶段 + 情绪稳定性 =====
    if narrative_beat == 'resolution' and total_rounds >= 15:
        stability = self._calculate_stability(emotion)
        if stability > 0.8:
            return True, "收束阶段情绪稳定", self._classify_ending(emotion)

    # ===== 渐进式结束 =====
    if total_rounds >= 15:
        prob = min(0.5, (total_rounds - 15) * 0.05)
        if random.random() < prob:
            return True, "渐进式结束", self._classify_ending(emotion)

    return False, "继续", None


def _classify_ending(self, emotion: EmotionState) -> str:
    """根据情绪状态分类结局"""
    pad = compute_pad_vector(emotion)
    intensity = (pad[0]**2 + pad[1]**2 + pad[2]**2) ** 0.5

    # 美好结局：高愉悦 + 高支配
    if pad[0] >= 0.3 and pad[2] >= 0.2:
        return "happy_ending"

    # 悲剧结局：低愉悦 + 高唤醒（痛苦但激烈）
    if pad[0] <= -0.3 and pad[1] >= 0.2:
        return "bad_ending"

    # 苦涩结局：低愉悦 + 低唤醒 + 低支配
    if pad[0] <= -0.2 and pad[1] <= 0 and pad[2] <= 0:
        return "bittersweet_ending"

    # 开放结局：其他
    return "open_ending"
```

## 四、AI 创作层（AICreator）

### 4.1 职责

AI 只负责创作，不负责决策：

| 任务 | 输入 | 输出 | 约束来源 |
|------|------|------|---------|
| 生成台词 | 节拍内容 + 玩家选择 + 角色性格 | 50-100 字台词 | 规则引擎提供节拍 |
| 生成选项文本 | 节拍的选项方向 + 情绪状态 | 3 个选项文本 | 规则引擎提供方向 |
| 生成事件规划 | 叙事锚点 + 情绪状态 | 草案结构 | 规则引擎提供锚点 |
| 生成结局 | 结局类型 + 情绪状态 | 结局故事背景 | 规则引擎判定类型 |

### 4.3 选项生成

```python
def generate_options(
    self,
    option_directions: list[str],
    current_emotion: EmotionState,
    character_name: str,
) -> list[PlayerOption]:
    """生成玩家选项（AI 填充文本，规则引擎预设变化量）"""

    # 规则引擎为每个方向预设 state_changes
    state_changes_presets = self.rule_engine.generate_option_state_changes(
        directions=option_directions,
        current_emotion=current_emotion,
    )

    prompt = f"""为玩家生成 3 个选项，每个选项 15-30 字。

选项方向：
1. {option_directions[0]}
2. {option_directions[1]}
3. {option_directions[2]}

当前情绪状态：好感度{current_emotion.favorability}，信任度{current_emotion.trust}

请直接输出 3 个选项，每行一个。"""

    options_text = self.llm.generate(prompt, max_tokens=100, temperature=0.8)

    # 组装选项（AI 文本 + 规则引擎的 state_changes）
    options = []
    for i, text in enumerate(parse_options(options_text)):
        options.append(PlayerOption(
            id=i,
            text=text,
            state_changes=state_changes_presets[i],  # 来自规则引擎
        ))

    return options
```

### 5.1 状态结构

@dataclass
class GameState:
    """游戏全局状态"""
    thread_id: str
    character_id: int
    scene_id: str

    # 情绪状态
    emotion: EmotionState

    # 剧本状态
    script_manager: ScriptManager
    rule_engine: RuleEngine
    ai_creator: AICreator

    # 进度追踪
    total_rounds: int = 0
    total_events: int = 0
    current_event_rounds: int = 0

    # 当前事件
    current_event: EventDraft = None
    current_beat_index: int = 0
    dialogue_history: list = []

    # 结局状态
    is_finished: bool = False
    ending_type: str = None
```

### 5.2 主循环

```python
def process_player_choice(self, option_id: int) -> dict:
    """处理玩家选择（主循环）"""

    # 1. 获取玩家选项
    option = self.current_options[option_id]

    # 2. 记录对话历史
    self.dialogue_history.append({'role': 'user', 'content': option.text})

    # 3. 规则引擎：计算情绪变化
    self.emotion = self.rule_engine.calculate_emotion_change(
        current=self.emotion,
        option_state_changes=option.state_changes,
        narrative_beat=self.current_event.narrative_beat,
        beat_emotion_target=self.current_beat.emotion_target_range,
    )

    # 4. 规则引擎：判定事件是否结束
    should_end, reason = self.rule_engine.should_advance_event(
        current_round=self.current_event_rounds,
        dialogue_history=self.dialogue_history,
        current_beat=self.current_beat,
        emotion=self.emotion,
    )

    if should_end:
        return self._handle_event_end(reason)

    # 5. AI：生成下一轮台词
    self.current_beat_index += 1
    next_beat = self.current_event.key_beats[self.current_beat_index]

    dialogue = self.ai_creator.generate_dialogue(
        beat=next_beat,
        player_last_choice=option.text,
        dialogue_history=self.dialogue_history,
        character_name=self.character_name,
        character_personality=self.character_personality,
        current_emotion=self.emotion,
    )

    # 6. AI：生成选项
    options = self.ai_creator.generate_options(
        option_directions=next_beat.option_directions,
        current_emotion=self.emotion,
        character_name=self.character_name,
    )

    # 7. 更新状态
    self.dialogue_history.append({'role': 'assistant', 'content': dialogue})
    self.current_event_rounds += 1
    self.total_rounds += 1

    return {
        'character_dialogue': dialogue,
        'player_options': [o.to_dict() for o in options],
        'current_states': self.emotion.to_dict(),
        'scene': self.current_event.scene,
        'is_game_finished': False,
    }


def _handle_event_end(self, reason: str) -> dict:
    """处理事件结束"""

    # 1. 生成事件摘要
    summary = self._create_event_summary()

    # 2. 存入记忆
    self.script_manager.completed_events.append(summary)

    # 3. 规则引擎：判定是否触发结局
    anchor_progress = self._calculate_anchor_progress()
    should_end, end_reason, ending_type = self.rule_engine.should_trigger_ending(
        total_rounds=self.total_rounds,
        total_events=self.total_events,
        emotion=self.emotion,
        narrative_beat=self.current_event.narrative_beat,
        anchor_progress=anchor_progress,
    )

    if should_end:
        return self._handle_ending(end_reason, ending_type)

    # 4. 推进剧本
    self.script_manager.advance_script(summary)

    # 5. 激活下一个事件
    self._activate_next_event()

    # 6. 生成新事件的第一轮台词
    return self._generate_first_round()
```

## 六、完整数据流

```
开局：
  玩家选角 + 选场景
       ↓
  ScriptManager.initialize_opening_event()
    → 从场景事件池随机选取开场事件（0 延迟）
    → 构建事件 0 的 KeyBeat[0]（轮次 1，预设内容）
       ↓
  ScriptManager.initialize_script()
    → 生成叙事锚点
    → 生成事件 1-3（简略版：标题+一句话描述）
       ↓
  使用预设内容展示事件 0 轮次 1 的台词+选项
  RuleEngine 生成选项的 state_changes
       ↓
  返回给玩家（首次响应 < 1 秒）

事件 0 轮次 1 结束后：
  玩家选择选项
       ↓
  RuleEngine.calculate_emotion_change()  ← 纯规则
       ↓
  AI 生成事件 0 轮次 2-N 的 KeyBeat（根据玩家选择）
  AI 生成轮次 2 台词
  RuleEngine 生成选项
       ↓
  返回给玩家

事件 0 轮次 2-N：
  玩家选择选项
       ↓
  RuleEngine.calculate_emotion_change()  ← 纯规则
  RuleEngine.should_advance_event()      ← 纯规则
       ↓
  [事件继续]                    [事件结束]
       ↓                              ↓
  AI 生成下一轮台词            创建事件摘要
  RuleEngine 生成选项          存入记忆
       ↓                              ↓
  返回给玩家                   RuleEngine.should_trigger_ending()
                                        ↓
                               [触发结局]          [继续]
                                    ↓                   ↓
                               AI 生成结局        ScriptManager.advance_script()
                               游戏结束            升级事件 1 为完整版
                                                   追加新的简略草案
                                                   AI 生成事件 1 第一轮台词
                                                   返回给玩家

事件 1-N：
  同事件 0 轮次 2-N 流程
```

## 七、AI 调用统计

| 操作 | 频率 | Token/次 | 说明 |
|------|------|---------|------|
| 生成叙事锚点 | 1 次 | ~300 | 开局 |
| 生成事件规划（简略） | 3 次 | ~100 | 开局 |
| 事件 0 剩余 KeyBeat | 1 次 | ~200 | 轮次 1 结束后 |
| 升级草案为完整版 | 每事件 1 次 | ~400 | 事件切换时 |
| 生成新简略草案 | 每事件 1 次 | ~100 | 事件切换时 |
| 生成台词 | 每轮 1 次 | ~150 | 每轮（事件 0 轮次 1 不需要） |
| 生成选项文本 | 每轮 1 次 | ~100 | 每轮（事件 0 轮次 1 不需要） |
| 生成结局 | 1 次 | ~300 | 结局时 |

**每局总成本估算**（假设 5 事件 × 3 轮 = 15 轮）：

```
开局：300 + 100×3 = 600 tokens（无事件 0 完整规划）
事件 0 轮次 1 后：200 tokens（生成剩余 KeyBeat）
事件切换：400×4 + 100×4 = 2000 tokens
台词+选项：(150 + 100) × 14 = 3500 tokens（事件 0 轮次 1 不需要）
结局：300 tokens
────────────────────────
合计：约 6600 tokens
```

vs 当前系统：约 4000-6000 tokens

**增加约 10-30%，但叙事质量大幅提升。事件 0 轮次 1 零延迟。**

## 九、风险与缓解
